## 1. 项目立项依据

### 1.1 研究背景

- 全球变暖背景下极端天气频发
- IPCC《Climate Change 2022》指出气候变化加剧农业系统脆弱性[1]
- **2023 年案例**：台风"杜苏芮"致河北临城县 3 天累计降水 1003.4 mm；海河流域 1963 年以来首次流域性洪水；玉米损失 22 万吨；2023 年全国平均气温 10.7℃ 创历史新高

**现有研究三大不足：**
1. **数据维度单一**：多依赖统计数据，缺少遥感、气象等多源融合
2. **模型可解释性不足**：难以精准识别核心风险因子并量化影响程度
3. **成果难落地**：多停留在算法层面，缺乏面向决策者的可视化工具

### 1.2 国内外研究现状

#### 国外研究

- 单一气候因子分析 → 多极端事件耦合
- **关键基础**：
  - Chen 等 (2016) **XGBoost** 梯度提升树 → 农业风险预测主流[2]
  - Lundberg 等 (2017) **SHAP** 值分解 → 可解释性统一框架[3]
  - Hochreiter 等 (1997) **LSTM** → 解决梯度消失，时序建模奠基[4]
- 多模态数据融合 + 深度学习兴起，但多基于发达国家，难适配发展中国家

#### 国内研究

- 紧扣"风险评估—影响机制—预警防控"
- 唐华俊 (2017)：极端气候是未来粮食生产核心威胁[5]
- 范振楠等 (2024)：绿色金融视角下极端气温影响有区域分化[6]
- 曹克苗 (2025)：XGBoost+SHAP 可解释风险预测[7]
- 黄鑫茜 (2025)：LSTM 黄淮冬小麦春季低温灾害监测[8]
- 李嘉乐 (2024)：深度学习大豆遥感估产归因[9]
- 田潭鹏 (2025)：黄河流域气候+农田水利耦合影响[10]

#### 三方面短板

1. 单一统计/遥感数据，**缺乏结构化、栅格化、时序化、地理化四类异构数据的统一融合**
2. 单一模型应用居多，**双重验证或多模型对比少**，难以确认结论稳健性
3. 算法成果与可视化工具脱节，**缺工程化平台**

### 1.3 研究意义

- **理论意义**：构建"非线性归因 - 时序验证"双模型协同框架；引入注意力机制使 LSTM 对极端事件更敏感
- **实践意义**：识别核心风险因子（前期实验为灌溉率、洪涝占比、日照时数）→ 差异化政策（低灌溉省份 → 节水灌溉建设；高洪涝省份 → 气象预警与排涝）
- **技术示范**：多模态数据融合 + 可解释 AI + 可视化系统集成的完整路径，可推广至水稻、小麦、病虫害风险

## 2. 项目内容、研究目标及关键科学问题

### 2.1 项目内容（四层架构）

四层架构：**数据层 - 模型层 - 业务逻辑层 - 表现层**

#### （1）多模态数据集构建

- **统计数据**：粮食单产、灌溉率、农机总动力、化肥施用量
- **气象数据**：年均气温、累计降水量、日照时数、SPEI 干旱指数
- **遥感数据**：MODIS NDVI 植被指数、地表温度（LST）
- **GIS 数据**：省级行政区划、耕地分布矢量
- **预处理**：清洗、线性插值、Butterworth 滤波去趋势、时空对齐、Z-score 标准化

#### （2）核心风险因子识别

- XGBoost 非线性风险预测
- SHAP 框架：全局重要性 + 局部决策 + 特征交互三维度

#### （3）双模型稳健性验证

- LSTM 时序模型对照验证
- **引入注意力机制（Attention Mechanism）** → 自适应聚焦关键时间节点
- 对比影响力排名与作用方向

#### （4）可视化决策系统开发

- **技术栈**：Vue + Flask
- **四大模块**：风险时空地图、SHAP 归因看板、参数情景模拟、韧性路径推荐

### 2.2 研究目标

| 目标 | 指标 |
|------|------|
| 模型精度 | 测试集 R² ≥ 0.62，RMSE ≤ 0.0055 |
| 系统响应 | 单次推理 ≤ 2 秒，普通 PC/云端可部署 |
| 核心因子 | 双模型交叉验证（前期：灌溉率、洪涝占比、日照时数）→ 31 省差异化韧性路径 |

### 2.3 关键科学问题

1. **多模态异构数据的时空对齐与融合**
   - 时间分辨率：年度 vs 逐日 vs 逐栅格
   - 空间口径：省级 vs 站点 vs 像元
   - 统一为"省份×年份×特征"面板格式

2. **核心因子识别结论的稳健性验证**
   - 两类原理不同模型交叉验证
   - 设计影响力排名一致性、作用方向一致性指标

3. **长期趋势与短期风险波动的分离**
   - Butterworth 低通滤波剥离技术进步分量
   - 提取气候波动的随机残差作为风险指标

## 3. 实施思路（四阶段）

### 3.1 第一阶段（2026.5 - 2026.7）数据采集与预处理

- **来源**：国家统计局、中国农村统计年鉴、CMDC、《中国水旱灾害公报》、NASA EARTHDATA
- **预处理**：
  - 线性插值缺失值
  - Butterworth 低通滤波 → 风险指标 Y
  - 栅格遥感按省域聚合 → 面板
  - 气象站点按省域加权
  - Z-score 标准化
- **产出**：31 省份 × 11 维特征 × 403 条样本标准化数据集

### 3.2 第二阶段（2026.8 - 2026.11）双模型构建与验证

- **XGBoost**：加法集成 K 棵树 + 正则化 + MSE；Grid Search（lr、max_depth、n_estimators）
- **SHAP**：基于 Shapley 值；全局重要性、局部决策、特征交互
- **Attention-LSTM**：遗忘/输入/输出门 + 注意力加权聚合 → 极端事件敏感
- **评价**：R² + RMSE；2022—2023 跨年度滚动验证

#### 前期初步成果

| 模型 | R² | RMSE |
|------|------|------|
| XGBoost | 0.6293 | 0.0051 |
| LSTM | 0.6619 | 0.0046 |

> **Top 3 因子（双模型一致）**：灌溉率 > 洪涝占比 > 日照时数

### 3.3 第三阶段（2026.12 - 2027.2）可视化系统开发

- **架构**：前后端分离
- **前端**：Vue3 + ECharts + Leaflet
- **后端**：Python Flask + RESTful API
- **数据库**：PostgreSQL
- **模型 API**：JSON 协议返回预测+归因
- **四大模块**：
  1. 风险时空地图（省域单元、历年演化）
  2. SHAP 归因看板（瀑布图、特征重要性图）
  3. 参数情景模拟（反事实预测）
  4. 韧性路径推荐（差异化政策）
- **部署**：云服务器 + 在线访问

### 3.4 第四阶段（2027.3 - 2027.4）成果整理

- 学科竞赛参赛
- 学术论文投稿 / 软著申请
- 项目代码、数据、文档
- 结题答辩
