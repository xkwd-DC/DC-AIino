## §1 动机:为什么必须做混合效应 PoC

`backend/models/strict_cv_v3_card.md` 已经实证:**random 8:2 R²=0.907,但 leave-province-out 中位 R²=-16.83**(31 省仅 1 省 R²>0)。strict CV 卡片用文字下了结论 ——"Essentially 100% of the headline 0.91 is attributable to learning province-level baselines" —— **但没有给出定量比例**。

混合效应模型 `y_it = α_i + β X_it + ε_it` 可以**显式吸收 α_i(省份 baseline)**,让 β 是纯 climate 边际效应。配套的 **Intraclass Correlation Coefficient (ICC)** 直接给出"省份固定效应占总方差的比例" —— 这是"R²=0.907 中省份记忆贡献多少 vs climate→yield 边际效应贡献多少"这个问题的**最终定量答案**。

这一 PoC 直接对应申报书 v6 §6.2(c) 与论文 §7.2(c) "未来工作"段已经写下的混合效应方向 —— 不再是空头支票,而是**结题前可兑现的方法学突破**。审阅人 §7.2 第 5 条已明确建议"M2/M3 阶段补做一次小规模 PoC,让未来工作有 6 个月内可兑现的下一步"。

## §2 数学公式 + 直觉

### 模型公式

$$y_{it} = \alpha_i + \beta^\top X_{it} + \epsilon_{it}, \quad \alpha_i \sim \mathcal{N}(\mu_\alpha, \sigma_\alpha^2), \quad \epsilon_{it} \sim \mathcal{N}(0, \sigma_\epsilon^2)$$

- $i \in \{1, \dots, 31\}$ 为省份索引,$t \in \{2011, \dots, 2023\}$ 为年份索引。
- $\alpha_i$:**省份级随机截距**(random intercept),吸收"河南天然单产高、青海天然单产低"这类**与气候输入无关的省份 baseline 差异**。
- $\beta$:**11 维固定斜率向量**(fixed slope),刻画 climate / 致灾 / 韧性投入对单产的**纯边际效应**,跨省共享。
- $\epsilon_{it}$:省份-年度残差噪声。

### 直觉对照

| 视角 | XGBoost 当前模型 | 混合效应模型 |
|---|---|---|
| α_i 处理方式 | 让模型从 Fert/Mech/Irr 等"指纹"特征**隐式学到**省份均值 | **显式列为参数估计**,与 X 分离 |
| β 的解释 | 与 α_i 纠缠,所以 SHAP 给出的是"指纹+边际"混合贡献 | 纯 climate→yield 边际效应,**可解释、可用于政策推演** |
| random vs fixed | XGB 无此区分 | $\alpha_i$ 作为 **random effect**(从总体分布抽样),$\beta$ 作为 **fixed effect**(总体参数) |

### 与 v3 11 维 X panel 的对接

- 输入特征严格沿用 `backend/models/feature_columns.json`:`Irr, Flood_R, Sun, Temp, SPEI, Prec, Mech, Fert, Drou_A, Flood_A, NDVI`(11 维)
- 目标:`yield_kg_per_ha`(与 v6 主报告口径一致)
- 数据源:`data/interim/paper_panel_v3.parquet`(403 行 × 27 列,**注意 parquet 中列名为小写** `irr, flood, sun, temp, spei, prec, mech, fert, drou_a, flood_a, ndvi`,代码里要 rename 一次对齐 feature_columns.json 大写口径或者反之)
- 分组键:`province`(31 个,parquet 中为中文省名,如"河北省")
- 标准化:与 XGB 一致,X 走 Z-score(可复用 `backend/models/scaler.pkl` 或重新 fit;若重新 fit 需在 card 里写明)

### 步骤 1:环境准备

- 不要装到全局 / 不要进 git。`pip freeze > /tmp/dc-mixed/requirements_snapshot.txt`,在 card 里贴关键版本号。

### 步骤 2:数据准备

```python
import pandas as pd
df = pd.read_parquet('data/interim/paper_panel_v3.parquet')
# 11 维 X(注意 parquet 列名为小写,这里 rename 对齐 feature_columns.json)
FEATURES = ['irr','flood','sun','temp','spei','prec','mech','fert','drou_a','flood_a','ndvi']
y = df['yield_kg_per_ha'].values
X = df[FEATURES].copy()
groups = df['province'].values   # 31 个
# Z-score(必须做,否则不同特征量级悬殊会让 β 估计方差炸)
X_z = (X - X.mean()) / X.std()
assert X_z.shape == (403, 11)
assert len(set(groups)) == 31
```

- 不要剔除任何行(N=403 已是结构上限,不容许在此 PoC 里再减)。
- 若发现某省份某年缺失,先在 EDA 单元报告,**不要静默 dropna**,在 card 里如实记录。

### 步骤 3:Fit 主模型(固定斜率 + 随机截距)

```python
import statsmodels.formula.api as smf
import statsmodels.api as sm
import numpy as np

data = X_z.copy()
data['yield'] = y
data['province'] = groups

# y ~ X + (1 | province)
formula = "yield ~ " + " + ".join(FEATURES)
md = smf.mixedlm(formula, data, groups=data['province'])
mdf = md.fit(method=['lbfgs'], reml=True)   # REML 是默认,但建议显式写
print(mdf.summary())
```

- 优化器:`lbfgs`(默认 `bfgs` 在 N=403 上偶尔不收敛,`lbfgs` 更稳)。
- REML(restricted maximum likelihood):用于 ICC / 方差成分估计的标准口径,默认即可。
- 若不收敛:① 检查 X_z 是否真正中心化、② 把不收敛特征做相关性热图剔除共线性极高的对(经验:Flood_R 与 Flood_A、SPEI 与 Prec 可能 |ρ|>0.9,在 card 里要标注是否处理过)。

### 步骤 4:LRT 检验(对比 pooled OLS baseline)

```python
# Pooled OLS baseline:y ~ X,无随机效应
ols = sm.OLS(data['yield'], sm.add_constant(data[FEATURES])).fit()
ll_ols = ols.llf
ll_mixed = mdf.llf
LR = 2 * (ll_mixed - ll_ols)
# 自由度 1(多估计了 σ_α^2 这一个方差成分)
from scipy.stats import chi2
p_value = chi2.sf(LR, df=1) / 2   # 边界 chi-bar-square,通常除 2(Self & Liang 1987 标准做法)
print(f"LRT statistic = {LR:.2f}, p ≈ {p_value:.4g}")
```

预期:LRT 统计量 >> 临界值,p < 0.001 —— 即"省份随机截距是显著有信息的方差成分",从而**统计上验证 v3 strict CV 卡片下"100% 来自省份 baseline"的文字结论**。

### 步骤 5:输出 α_i 估计 + β estimate + ICC

```python
# α_i 估计(31 省 baseline 偏移,相对于总体均值 μ_α)
alpha_i = mdf.random_effects  # dict: province -> 1-D array(只有 1 维随机截距)
alpha_table = pd.DataFrame(
    [(p, v.iloc[0]) for p, v in alpha_i.items()],
    columns=['province', 'alpha_hat']
).sort_values('alpha_hat')

# β estimate + CI(95%)
beta_table = pd.DataFrame({
    'feature': FEATURES,
    'beta_hat': mdf.fe_params[FEATURES].values,
    'se':       mdf.bse_fe[FEATURES].values,
    'p_value':  mdf.pvalues[FEATURES].values,
    'ci_lo':    mdf.conf_int().loc[FEATURES, 0].values,
    'ci_hi':    mdf.conf_int().loc[FEATURES, 1].values,
})

# ICC = σ_α^2 / (σ_α^2 + σ_ε^2)
var_alpha = float(mdf.cov_re.iloc[0, 0])
var_eps   = float(mdf.scale)
icc = var_alpha / (var_alpha + var_eps)
print(f"ICC = {icc:.4f}  (预期 0.85+,意味着省份固定效应主导)")
```

输出物:`alpha_table`(31 行,排序后存 CSV)、`beta_table`(11 行,含 95% CI)、ICC 单值 + 95% CI(可用 bootstrap 200 次或 statsmodels profile likelihood)。

## §6 资源需求 + 风险

### 6.1 资源需求

- **算力**:CPU 足够。statsmodels MixedLM 在 N=403 × 11 维上单次 fit 通常 < 1 分钟。即使加 bootstrap 200 次估 ICC CI,总时间 < 5 分钟。**无需 GPU,无需远程节点**。
- **存储**:卡片 + json + csv + notebook 总计 < 1 MB。
- **依赖**:仅 statsmodels + pandas + pyarrow + numpy + scipy。装到 `/tmp/dc-mixed`,不入 git。
- **时间预估**:数据准备 0.5h + fit + 调通 1.5h + 写卡片 2h + LRT/ICC bootstrap 1h = **共约 0.5–1 天**(熟练度高时半天可完成)。

### 6.2 主要风险

1. **固定斜率假设可能过强**:当前模型 β 跨省共享。某些特征(如 SPEI 在西北 vs 华南可能斜率方向相反)可能违反此假设。
   - **缓解**:在 notebook 末尾跑一次 random slope 对比模型 `y ~ X + (1 + SPEI | province)`(挑 1-2 个气候核心特征,不要全 11 维都加 random slope,会发散),通过 LRT 检验是否需要扩展。**如时间紧,在 card 的"局限"段写明该假设即可,留作 v2 PoC**。
2. **收敛性**:小样本 + 11 维 + 31 组,statsmodels 有时报"Singular matrix"。
   - **缓解**:① 用 `method=['lbfgs']` 或 `['cg']`、② 剔除明显共线性极高的对(Flood_R/Flood_A 二选一可尝试)、③ 在 cards 里如实记录尝试过哪些设定;**不要为了让模型跑通而偷偷加 ridge / 改数据**。
3. **ICC 解读边界**:ICC 是"省份固定效应方差占比",不等于"R²=0.907 中省份贡献占比"的严格分解。
   - **缓解**:在 card 里写清"ICC 在概念上对应'省份固定效应方差成分占比',与 XGB R²=0.907 的关系是定性对应而非严格 100% 等价分解"。这一点诚信交代,反而显严谨。
4. **NDVI 共线性**:NDVI 与 LST / 气温有较强相关性,在固定效应回归下 β_NDVI 可能不显著 —— 这与 v3 NDVI ablation R² 仅降 0.003 的方向一致,**反而是好结果**,在 card 里强调这一互证。
