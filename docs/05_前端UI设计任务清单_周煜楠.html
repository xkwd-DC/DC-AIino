SECTION · 01

一、你要先理解的三件事

1. 你做的不是"完整网站"，是"设计稿"

你做出来的每一份 HTML 都是给潘妙齐参考的视觉蓝本。他会把你的设计在 Vue 工程里重新实现一遍，所以：

✓ 你需要做的
只做单页 HTML，不考虑路由、API、数据库；数据全部写死在代码里（mock 数据），不调真模型；视觉、交互、布局做漂亮就行。
✗ 你不用管的
不需要懂 npm、git、终端命令；不用考虑响应式适配；不用考虑数据来源；不用学 Vue。

2. claude.ai 的 Artifact 是你的核心工具

每次发起对话，Claude 会在右侧生成一个 Artifact（HTML/React 文件），你可以：

  • 预览：右上角 Preview 切换到效果图
  • 复制代码：右上角 Copy 复制完整代码
  • 下载：点击下载按钮存为 .html 文件
  • 分享：点击 Publish 生成可分享链接

3. 设计风格已经定了——不要再发散

我们的项目是农业 + 政府决策支持性质，不是消费类 App，所以：

用不用
深绿色为主调（农业气质）渐变紫、霓虹色（科技感过头）
深色背景（专业、严肃）Bootstrap 默认样式
Noto Serif SC（标题）Comic Sans、Arial
JetBrains Mono（数字）Emoji 装饰
数据密度高大段空白配大字
克制的微交互满屏动画

三、任务 1：建立设计系统（半天，最重要！）

⚠ 为什么先做这个？
4 个模块的视觉风格必须一致，否则潘做出来的成品像 4 个不同的网站。设计系统是给后面 4 个模块当"配色字典"用的。
TASK 01 半天 设计系统 design-system.html

任务目标

输出一份 design-system.html，包含：

  • 完整的颜色变量定义（CSS variables）
  • 字体规范（标题、正文、数据三套）
  • 8 种常用组件样例（卡片、按钮、滑块、表格、标签、徽章、图表配色、空状态）
  • 间距与圆角规范

直接复制下面这段 Prompt 发给 Claude

Prompt · 设计系统初始化
我在做"基于多模态的极端气候下我国粮食生产风险核心因子识别与韧性提升可视化系统"项目，需要你为我建立一份设计系统（Design System）HTML 文档。

【项目调性】
- 服务对象：农业管理部门、农业保险机构、农业科研机构
- 视觉气质：政务+学术+科技感的交叉，要让评委一眼觉得"专业可信"
- 灵感参考：Stripe Dashboard、Linear、Vercel Analytics 的克制美学，但配色用农业绿（不要科技蓝紫）

【硬性要求】
1. 深色主题（dark mode），主背景 #0A1612 到 #0F1F1A 之间
2. 主色用深绿系：#1B4332 / #2D6A4F / #74C69D / #95D5B2
3. 警示色用暖红：#E07856 / #B14A36（用于风险高的省份）
4. 中性色用棕灰：#E8E4D8（文字）/ #A8B5AC（次文字）/ #6B7A72（弱化）
5. 强调色用琥珀：#E6B655（用于中等风险、数字高亮）
6. 字体三套：
   - 标题：Noto Serif SC（衬线，让中文有"报告感"）
   - 正文：Noto Sans SC
   - 数字/代码：JetBrains Mono
7. 圆角整体克制：卡片 4-6px，按钮 4px，徽章 2-3px
8. 阴影几乎不用，靠 border + 背景层次区分

【需要包含的组件样例】
请把下面每个组件都做一个真实样例放在页面上：

A. 颜色系统：色板可视化，每个色显示色值、用途
B. 字体规范：H1-H6 + 正文 + 数字三套，给真实示例文字
C. 卡片组件：3 种变体（普通、带标题、带 meta）
D. 按钮组件：主按钮、次按钮、危险按钮、禁用态、加载态
E. 滑块（slider）：单值、双值范围、带刻度的滑块
F. 表格：紧凑型数据表，有 zebra stripe，有数字右对齐
G. 徽章/标签：风险等级标签（低/中/高/极高）、状态标签（正常/警告/错误）
H. 图表配色卡：给出 5 种数据可视化色板（顺序色阶、发散色阶、定性色板）
I. 空状态：当某个模块没有数据时怎么显示

【交互细节】
- 所有 hover 状态都要有，过渡 0.2-0.25s
- 滑块拖动要顺滑
- 不要任何 emoji 装饰
- 可以用极简的 SVG 图标（如箭头、放大镜、铃铛）但不超过 8 个

【输出要求】
1. 单文件 HTML，CSS 全部用 :root CSS variables 定义
2. 顶部加一个目录导航（锚点跳转）
3. 每个组件下方加一段说明：什么时候用、什么时候别用
4. 不需要 JavaScript，纯静态展示页

做完先给我看效果，我会反馈调整。
    验收标准
  • 整体看上去像"政府/科研机构的设计规范网站"，不是消费 App
  • 颜色全部走 CSS 变量，没有硬编码色值
  • 有 8 类组件，都能看到真实样例
  • 字体规范清晰，中文不糊
  • 没有任何 emoji，没有渐变紫
如果不满意，直接告诉 Claude 哪里不对

例如："色板太暗了，把 #0A1612 提亮到 #0D2118"
"卡片圆角太大了，从 6px 改到 4px"
"字体规范没有给真实示例文字，每个号都要写一段例子"

四、任务 2：风险时空地图模块（1-2 天）

★ 模块定位
这是整个系统的门面，评委一进来第一眼看到的就是这个。要做到一打开就"哇"。

数据规范

地图上展示 31 省的粮食生产风险值（去趋势单产波动 Y），数据区间 0.011 - 0.043。Top-3 高风险省份是青海(0.040)、西藏(0.038)、新疆(0.036)，低风险是北京(0.018)、上海(0.019)、黑龙江(0.020)。

Prompt · 风险时空地图
基于我项目知识里的设计系统，请帮我做"风险时空地图"模块的高保真 UI 原型，单文件 HTML，用 ECharts 5（CDN 引入）。

【布局】
- 顶部：模块标题、副标题、右侧实时推理状态徽章
- 左侧 70%：中国地图 + 底部年份滑块
- 右侧 30%：图例 + Top10 高风险省份列表 + 当前 hover 省份详情卡

【中国地图要求】
- 用 ECharts registerMap 加载 china.json（从
  https://cdn.jsdelivr.net/gh/apache/echarts@4.6.0/map/json/china.json
  获取）
- 31 省按风险值上色，色阶用设计系统的 visualMap：
  绿 → 浅绿 → 琥珀 → 暖红 → 深红
- 省份 label 默认半透明显示，hover 时变白色加粗
- 支持鼠标滚轮缩放、拖动平移
- 背景透明（融入页面深色背景）

【年份滑块】
- 范围 2011-2023
- 滑块拖动时地图数据实时更新（用伪随机数模拟年度波动，每年的省份排名应该有合理变化但不要完全打乱）
- 滑块旁边显示当前选中年份（用 JetBrains Mono 字体，大号）

【Top10 列表】
- 按风险值降序，序号用 01-10 两位数（JetBrains Mono）
- 每行：序号、省份名、风险值
- 风险值用颜色编码：>0.030 红、0.025-0.030 琥珀、<0.025 绿
- 点击某行时，地图自动高亮对应省份

【明细卡】
- 默认显示"悬停省份查看明细"占位
- hover 地图时实时更新，显示：
  - 省份名（大号衬线字体）
  - 风险类型标签（如"洪涝主导型"、"旱作主导型"等）
  - 5 个关键指标：风险指数 Y、灌溉率%、洪涝占比%、日照时数 h、SPEI 干旱指数
  - 每个指标用 progress bar 横向条显示相对全国位置

【交互要求】
- 所有过渡 250ms ease
- 滑块拖动有惯性
- 地图 hover 有 200ms 防抖
- 整个页面加载时有 stagger 入场动画（每个卡片间隔 80ms 淡入上浮）

【数据】
请在 JS 顶部定义一个 PROVINCES 对象，包含 31 省的 mock 数据：
- 省份名（中文简称：北京、河北、山西…）
- y（风险值，0.011-0.043 区间）
- irr（灌溉率，20-120）
- flood（洪涝占比，0-24）
- sun（日照时数，970-3350）
- spei（干旱指数，-3 到 3）
- temp（年均温，2-26）
- type（风险类型标签，如"洪涝主导型"、"旱作主导型"、"沿海台风型"等）

数据要符合现实：南方省份洪涝占比高、西北日照多但 SPEI 偏干、东北温度低、新疆/青海风险值高。

【兜底】
如果中国地图 JSON 加载失败，地图区域显示一个"地图加载中..."的占位 SVG，不要白屏。

做完先给我看效果。
    验收标准
  • 中国地图正常显示，31 省都有颜色
  • 年份滑块拖动时数据有变化
  • hover 省份时右侧明细卡实时更新
  • Top10 列表准确反映当前数据
  • 整体视觉和设计系统统一
  • 在 1920×1080 屏上效果完美，在 1440×900 笔记本屏也不溢出

进阶迭代（如果还有时间）

  • 省份对比模式（选 2 个省份并排显示）
  • 风险变化趋势小图（hover 时显示该省 13 年趋势 mini chart）

五、任务 3：SHAP 归因看板模块（2-3 天）

★ 模块定位
这是项目"可解释 AI"的核心展示位，专业评委会盯着这里看。要做到学术规范 + 视觉惊艳。

数据规范

11 个特征的全局 SHAP 重要性（基于论文实测）：

排名特征重要性方向
1灌溉率 Irr0.420韧性（减损）
2洪涝占比 Flood_R0.385致灾（增灾）
3日照时数 Sun0.312韧性
4平均气温 Temp0.225致灾
5降水量 Prec0.185致灾
6干旱指数 SPEI0.162韧性
7农机总动力 Mech0.124韧性
8化肥施用量 Fert0.092致灾
9旱灾面积 Drou_A0.075致灾
10水灾面积 Flood_A0.058致灾
11NDVI 异常0.034韧性
Prompt · SHAP 归因看板
基于设计系统，请帮我做"SHAP 归因看板"模块的高保真 UI 原型，单文件 HTML，ECharts 5。

【布局】2×2 网格
- 左上：全局特征重要性柱状图（Mean |SHAP|）
- 右上：单样本预测分解瀑布图（SHAP Waterfall）
- 左下：SHAP 蜂群分布（Beeswarm Plot）
- 右下：核心因子自然语言解读卡片列表

【全局特征重要性柱状图】
- 横向条形图（h-bar），11 个特征
- 按重要性降序（从上到下：灌溉率、洪涝占比、日照时数...）
- 颜色编码：韧性因子用绿色渐变，致灾因子用红色渐变
- bar 末端显示数值（保留 3 位小数）
- 标题：「全局特征重要性 — Mean |SHAP value|」
- 副标题：「基于 XGBoost-SHAP 框架，n=403」

【SHAP Waterfall】
- 主题：「单样本预测分解 — 河南省 2022 年度」
- 起点：E[f(x)] 基线 = 0.0235
- 终点：f(x) 预测值 = 0.0249
- 中间 6 个特征贡献：
  + 洪涝占比 4.2% → +0.0028
  + 化肥施用量 → +0.0009
  + 平均气温 14.2°C → +0.0015
  − 灌溉率 65.4% → −0.0022
  − 日照时数 2240h → −0.0011
  − 农机总动力 → −0.0005
- 正贡献用红色（推高风险），负贡献用绿色（降低风险）
- 基线和终点用琥珀和蓝色高亮区分

【SHAP 蜂群图】
- Y 轴：Top 5 特征（灌溉率、洪涝占比、日照时数、平均气温、SPEI）
- X 轴：SHAP 值，0 为分界，左减损右增灾
- 每个特征 50-80 个点，X 位置 = SHAP 值，Y 位置 = 特征位置加抖动
- 点颜色 = 该样本下特征的原始值（用 visualMap 蓝→红梯度）
- 用 ECharts scatter，symbolSize: 5, opacity: 0.7

【自然语言解读卡片】
做 4 张卡片，每张：
- 顶部：图标（↑或↓）+ 因子名 + 韧性/致灾标签
- 中部：一句话核心结论
- 底部：一段 2-3 行的详细解释（用真实数字）

例如灌溉率卡片：
图标 ↓ | 灌溉率 (Irr) | 韧性因子
"高灌溉率显著负向拉动风险值"
"是抵御极端气候的核心人为调节手段。SHAP 平均贡献 −0.42，全国均值 56.7%，提升至 70% 可降低单产风险约 15%。"

四张卡片分别是：灌溉率、洪涝占比、日照时数、"灌溉率×气温"交互效应。

【上方控件栏】
顶部放一个 segmented control，允许切换：
- 全部样本（默认）
- 仅北方省份
- 仅南方省份
- 仅主产区（13 个粮食主产省）

切换时柱状图和蜂群图的数据有合理变化（可以用伪数据，但比例关系要保持现实感）。

【交互】
- hover 柱状条显示完整 tooltip
- 点击柱状条切换瀑布图的特征（如点击"平均气温"则瀑布图变成气温在该样本的局部解释）
- 蜂群图点 hover 显示该样本的省份+年份+特征值
- 卡片 hover 时整张卡片轻微上浮（translateY(-2px) + shadow）

数据 mock 即可，比例要符合论文表格。
    验收标准
  • 四张图严格按 2×2 布局，没有错位
  • 柱状图颜色编码清晰（绿=韧性，红=致灾）
  • Waterfall 起点终点 + 6 个中间贡献都标出
  • 蜂群图看起来像真实 SHAP 输出（点的颜色梯度合理）
  • 解读卡片文字专业、不空话
SECTION · 06

六、任务 4：参数情景模拟模块（1-2 天）

★ 模块定位
这是项目的"可玩性"亮点，评委会上手拖滑块。要做到即时反馈 + 视觉惊喜。
Prompt · 参数情景模拟
基于设计系统，请帮我做"参数情景模拟"模块的高保真 UI 原型，单文件 HTML，ECharts 5。

【布局】1:1.5 两栏
- 左侧 40%：参数控制台（省份下拉 + 5 个滑块 + 重置按钮）
- 右侧 60%：结果展示（双仪表盘 + 风险变化大数字 + 因子贡献对比图）

【左侧参数控制台】
1. 省份下拉框：31 省，默认"河南"
2. 5 个滑块（每个滑块独立卡片）：
   - 灌溉率（20-120%，默认根据省份基线）
   - 洪涝占比（0-24%）
   - 日照时数（970-3350 小时/年）
   - 平均气温（2-26°C）
   - SPEI 干旱指数（-3 到 +3）
3. 每个滑块卡片包含：
   - 顶部：参数名 + 当前值（JetBrains Mono 大号绿色）
   - 中部：滑块本体
   - 底部：刻度标签（最小值、方向提示、最大值）
4. "重置参数为省份基线值"按钮（次按钮样式）

【右侧结果展示】
1. 顶部双仪表盘并排：
   - 左仪表盘：基线风险（琥珀指针）
   - 右仪表盘：模拟风险（绿/红指针，根据是否改善）
   - 仪表盘色带：绿→琥珀→红，刻度 0.005-0.055
2. 中间「风险变化 Δ」大数字卡：
   - 标签：「风险变化 Δ」
   - 主体：超大数字（36px JetBrains Mono），如 "−0.0028"
   - 改善时显示绿色，恶化时显示红色
   - 副标题：「较基线风险降低/上升 X%」
3. 底部因子贡献对比柱状图：
   - 5 个特征的 SHAP 贡献横向条形图
   - 当前调节的因子贡献条会高亮（粗一点 + 发光描边）

【实时计算逻辑】
当任意滑块变动时，按下面线性近似公式实时计算（不要太精确，看起来合理就行）：

baseRisk = 0.0235  (E[f(x)])
irrEffect = -(irr - 56.7) * 0.000115
floodEffect = (flood - 2.97) * 0.000420
sunEffect = -(sun - 2086) * 0.0000023
tempEffect = (temp - 14.0) * 0.000180
speiEffect = -(spei - (-0.08)) * 0.000310
simRisk = base + 所有 effect 之和
delta = simRisk - 省份基线 Y

【交互细节】
- 滑块拖动时仪表盘指针平滑过渡（不要瞬间跳）
- 大数字卡有 count-up 动画（从旧值动到新值）
- 调节滑块时对应的贡献条短暂闪烁（如金黄色边框 300ms）
- 顶部加一个小型"近期模拟历史"，记录最近 5 次的模拟参数和结果（像 Linear 的 Recent Actions）

【右上角加】
"另存为情景"按钮：点击后弹出一个保存对话框（mock，不需要真存），记录情景名+当前所有参数值

数据 mock，公式如上。
    验收标准
  • 5 个滑块都能正常拖动，数值实时更新
  • 仪表盘指针平滑过渡
  • 大数字卡有 count-up 动画

十、常见问题

Q1：Claude 生成的代码太长，预览跑不起来？

发送"代码有没有语法错误？请检查并修复，告诉我修了什么"。

Q2：中国地图加载失败？

网络问题或 CDN 被墙。让 Claude "改用 https://geojson.cn/api/china/100000.json 这个备用源"。

Q3：图表颜色不在设计系统色板里？

让 Claude "所有图表配色严格走 CSS 变量，不要硬编码 hex 值"。

Q4：响应式怎么处理？

你不用管，重点做 1920×1080 桌面端就行，潘后续会做响应式适配。

Q5：能不能用 React 而不是 HTML？

能。在第一句加上"用 React + Tailwind 实现"。但 HTML 更省心，建议 4 个模块都用 HTML。

Q6：和潘对接时给什么？

给他 HTML 文件 + 截图 + 一句话说明（"这是 M01 风险地图，右侧明细卡的设计已经定稿，请按此实现 Vue 版"）。
