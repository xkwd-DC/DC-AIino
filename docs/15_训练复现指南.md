### §1.1 系统要求

| 项 | 最低 | 推荐 |
|---|---|---|
| OS | Linux x86_64(Ubuntu 22.04 验证) | Ubuntu 22.04 LTS |
| Python | **3.10.x** (硬要求,见 `.python-version`) | 3.10.20 |
| RAM | 4 GB(CPU 推理) | 8 GB(训练 LSTM 较舒服) |
| Disk | 2 GB 装依赖 + ~500 MB artifact + ~100 MB 数据 | — |

| GPU | 不要求(`tensorflow-cpu` 走 CPU);LSTM/Att-LSTM 总训练 ~5-8 min 可接受 | NVIDIA + CUDA 11.8(可选;走 `tensorflow` 包替换 `tensorflow-cpu`,改 requirements.txt) |

### §1.2 为什么硬钉 Python 3.10

- 训练 artifact 由 keras 3.14(写时)产出,Dense 层 config 带 `quantization_config`。
- Python 3.10 上 pip 装的 keras 上限是 3.12,加载报 `Unrecognized keyword arguments passed to Dense: {'quantization_config': None}`。
- 修复:`backend/api/predict.py:_apply_keras_compat_shim()` 在加载前 monkey-patch `Dense.from_config` 丢弃 unknown kwarg。
- 这个补丁在 Python 3.10 + keras 3.12 上验证过(Issue #39);其它组合(3.11 + keras 3.13 / 3.14)未验证,**不保证可加载**。

### §1.5 CPU 可复现性约束

10 种子均值在 CPU 上是稳定的,**但单个 seed 的 R² 可能跨平台漂移**(TF cuDNN 非确定算子、numpy BLAS 后端、scikit train_test_split CRT 实现差异),所以:

- **不要** 拿单 seed 数字比较平台间差异。
- **要** 拿 10 种子 mean ± std 比较 — 详见 §4 的 tolerance 区间。

## §3 一键 retrain(3 模型 × 10 种子)

### §3.1 全流程

```bash
# 默认:从 git root 跑,10 种子 0..9
bash scripts/retrain_all.sh
```

脚本会:
1. 校验 `data/interim/paper_panel_v3.parquet` 的 SHA-256(不匹配只 warn 不拦)
2. 顺序训练:`train_xgb_baseline.py` → `train_lstm_baseline.py` → `train_att_lstm_baseline.py`
3. 每个模型的 stdout 落到 `reports/retrain_<timestamp>/<model>.log`
4. 训练完聚合 3 个 `backend/models/*_seeds_results.json` → `reports/seeds_results.csv`
5. 跑 `python -m services.inference` 推理 sanity check(河南 2022 → 真值 4615 kg/ha)

预计耗时(CPU,2 vCPU 4 GB):
- XGB:**~1-2 min**(10 seeds × SHAP TreeExplainer)
- LSTM:**~5-8 min**(10 seeds × 100 epochs × EarlyStopping)
- Att-LSTM:**~5-8 min**
- 总计:**~15-20 min**

### §3.2 子集训练 / 跳过模型

```bash
# 只跑 XGB(LSTM/Att-LSTM 不动)
SKIP_LSTM=1 SKIP_ATT_LSTM=1 bash scripts/retrain_all.sh

# 自定义 PY_BIN(若 /tmp/dc-infer 不存在)
PY_BIN=/usr/bin/python3.10 bash scripts/retrain_all.sh
```

### §3.3 SEEDS 环境变量(v1 仅记录,v2 启用)

```bash
SEEDS=0,1,2 bash scripts/retrain_all.sh
```

⚠️ **当前 v1 行为**:`train_*.py` 顶部硬编 `SEEDS = list(range(10))`。`SEEDS` env var 只在 wrapper 日志里记录,**不传入训练脚本**。要精确控制种子,直接 patch 训练脚本(单独 commit,不要混进文档 PR)。

未来 v2 改进路径:训练脚本接受 `argparse` 或 env var `DC_SEEDS`,wrapper 透传(超出 T6 范围,留 G5 / future PR)。

## §4 预期 R²(±tolerance)

> 协议:**random 8:2 + 10 seeds**(论文协议;`docs/07_模型训练任务_算法组.md` §3.1 硬指标)。
> **不是** 三协议鲁棒性数字(那个在 §6)。

| 模型 | test R² mean | test R² std | test RMSE (kg/ha) | 来源 model card |
|---|---|---|---|---|
| XGB | **0.9072** | 0.0383 | 312.5 | `backend/models/model_card.md` |
| LSTM | **0.8856** | 0.0223 | 362.0 | `backend/models/lstm_model_card.md` |
| Att-LSTM | **0.8160** | 0.0502 | 456.4 | `backend/models/att_lstm_model_card.md` |

### §4.1 复现 tolerance(平台间漂移区间)

10 种子均值在 CPU/GPU/不同 BLAS 后端上**应当落在以下区间**:

| 模型 | 期望 R² mean 区间 |
|---|---|
| XGB | [0.87, 0.94](±2 std = ±0.077,但实测漂移极小) |
| LSTM | [0.86, 0.91] |
| Att-LSTM | [0.77, 0.86] |

**超出区间**说明环境有问题(numpy/sklearn 版本不对、TF 后端非确定、数据 hash 不对)。先按 §7 排错。

### §4.2 ⚠️ 关于这个 0.9072 的诚实诠释(v6 叙事守门)

**这不是"高精度跨省份预测"。** 详见 `backend/models/strict_cv_v3_card.md`:

- random 8:2 同省份训测混合,模型靠 "31 省 baseline 记忆 + 时间信号" 拿到 0.9072
- **NDVI 移除后 R² 几乎不变**(Δ = -0.003),NDVI 不是预测核心驱动
- **leave-province-out 中位 R² = -16.83**(31 省仅 1 省 R²>0),空间外推完全失败

→ 0.9072 应理解为 "模型对 31 个已知省份的 baseline 表征能力",**不是** "climate → yield 通用映射"。本研究的对外叙事(申报书 / 论文 / 答辩 / 路演)按 v6 基线(`docs/_craic/04_申报书_v6_rewrite_draft.md`)走。

## §6 三协议鲁棒性回归

> 本节是国家级评审 / 论文 reviewer / 中期检查最该看的部分。**诚实披露 leave-province-out 中位 -16.83**,体现项目对 v6 叙事守门的执行。

### §6.1 协议定义

| 协议 | 名称 | 切分方式 | 测什么 |
|---|---|---|---|
| A | **Random 8:2** | 行级随机 80/20,10 seeds | 论文协议(同省份训测混合) |
| B | **Leave-Year-Out** | 13 fold,每 fold 留一年作 test(31 行/fold) | 时序泛化(已知省 + 新年份) |
| C | **Leave-Province-Out** | 31 fold,每 fold 留一省作 test(13 行/fold) | 空间泛化(新省份预测) |
| D | **Stratified 5-fold** | 5 fold,row 级 mixed | 5-fold 对照 |

### §6.2 已落地结果(`backend/models/strict_cv_v3_results.json` + `strict_cv_v3_card.md`)

| 协议 | R² mean | R² 其他 | RMSE mean | n_folds |
|---|---|---|---|---|
| **A · Random 8:2** | **0.9072** | std=0.038, max=0.9636 | 312.5 | 10 |
| **B · Leave-Year-Out** | **0.8943** | std=0.104, min=0.6234 | 296.4 | 13 |
| **C · Leave-Province-Out** | **-176.85**(均值被极端值拉爆) | **median=-16.83**, q25=-74.7, max=0.173, **n_positive=1/31** | 825.0 | 31 |
| **D · Stratified 5-fold** | 0.9091 | std=0.027 | 311.2 | 5 |

### §6.3 怎么复现这些数字

⚠️ **当前 git 仓内 v1 训练脚本(`train_xgb_baseline.py` 等)只跑协议 A**(random 8:2,10 seeds)。
协议 B/C/D 的结果由熊鑫 5/22 在分支脚本上产出,JSON / card 已 commit 入 `backend/models/`,**但产生它们的脚本尚未规范化到 git**(docs/13 §0.2 G4 余项)。

**复现协议 A**(完全在 git 内):


**复现协议 B/C/D**(临时方案,直到 G4 规范化脚本入 git):
- 直接看 `backend/models/strict_cv_v3_results.json`(31 省 + 13 年完整明细已存盘)
- 等熊鑫的 `train_strict_cv_v3.py` 落地为 `scripts/` 入口后,本指南 §6.3 升级为一键命令
- 中期检查 / 论文 reviewer 可直接引用 JSON,不必要求重跑

### §6.4 协议 C 期望区间(诚实标注)

按 `strict_cv_v3_card.md`:

## §7 故障排除(常见坑)

### §7.1 Keras `quantization_config` 加载失败(Issue #39)

**症状**:
```
ValueError: Unrecognized keyword arguments passed to Dense: {'quantization_config': None}
```

**原因**:训练 artifact 由 keras 3.14 保存,Dense config 带 `quantization_config`;Python 3.10 上 pip 装的 keras 上限 3.12 不识别。

**修复**:已在 `backend/api/predict.py:_apply_keras_compat_shim()` 提供 monkey-patch。
- 用 `python -m backend.api.predict` 或走 Flask 入口会自动应用
- 手动调用 inference 模块时,在 `tf.keras.models.load_model()` 前 先 import 一次 `_apply_keras_compat_shim()` 并调用

**根治方案**(未来 v2):升级 Python 3.11 + keras 3.14,删 shim。

### §7.2 LSTM 推理输出离谱小(|y| < 10)

**症状**:`predict_lstm_yield()` 返回 ~ 2.5 kg/ha 而非 ~ 4615。

**原因**:漏调 `y_scaler.inverse_transform()`。LSTM 训练时 y 做了 StandardScaler 归一化(因为 LSTM 梯度优化对 kg/ha 3000+ 量级初始 MSE ~1.3e7 不收敛),predict 返回 z-score,必须 inverse_transform 回 kg/ha。

**修复**:`backend/services/inference.py:predict_lstm_yield()` 已封装。如果你写新的 inference 入口,**必须** 加载 `lstm_y_scaler.pkl` 并 inverse_transform。

### §7.3 `feature_columns.json` 列名映射错(PR #21 / Issue #24)

**症状**:训练 ↔ 推理 ↔ SHAP plot 三处列名不一致,SHAP 图缺列 / 错位。

**根因**:parquet 用小写(`flood`),训练 / inference 用大写(`Flood_R`),`scripts/plot_shap_v3.py` 早期版本漏映射。

**修复**:已在 `scripts/plot_shap_v3.py` 落地 `flood_r → flood` 别名映射 + `compile=False` 加载 Keras h5。任何新写的脚本必须复用 `backend/services/inference.py:API_TO_TRAINING` 映射常量,**不要** 手写。

### §7.4 `tf.keras` import 时报 CUDA 警告

**症状**(WARN,不影响功能):
```
... could not open libcudnn.so.8 ...
... could not open libcusolver.so.11 ...
```

**原因**:`tensorflow-cpu` 是 CPU-only 包,但 TF 启动时仍探测 GPU 后端。

**修复**:不需要修,这些是 WARN 不是 ERROR。要禁掉:
```bash
export TF_CPP_MIN_LOG_LEVEL=2     # ERROR 以下不打
export TF_ENABLE_ONEDNN_OPTS=0    # 关 oneDNN 数值微差
```
(训练脚本顶部已 setdefault 这俩)

### §7.5 SHAP 装不上(Python 3.14 / numpy 2.x 不兼容)

**症状**:`pip install shap` 在 Python 3.14 / numpy 2.2 上编译失败 / 加载报错。

**原因**:SHAP 0.46 依赖 numba,numba 在 3.14 / numpy 2.2 上滞后。

**修复**:本项目硬钉 Python **3.10** + numpy **1.26**,这两个约束守住 SHAP 装得上。如果你在 3.11+ 上不能装 SHAP,**可以临时 skip**(改 `train_xgb_baseline.py` 注释掉 SHAP 段),R² 数字不变,但拿不到 SHAP top-5。Att-LSTM attention 权重不受影响。

### §7.6 训练中断 / EarlyStopping 提前停

**症状**:`epochs=87` 而不是 100,看着像中断。

**说明**:这是 **正常行为**,不是 bug。`EarlyStopping(patience=10)` 在 val_loss 10 epoch 不下降时回滚到 best weights + 停止。`lstm_model_card.md` 各 seed 明细里 `epochs_run` 字段记录了真实 epochs;复现时跨平台 ±5 epochs 差异属正常。

### §7.7 `data/interim/paper_panel_v3.parquet` hash 不匹配

**可能原因**:

- 重训 R² 会偏离本指南 §4 期望值,**这是预期的,不是 bug**。

### §8.2 model card 自动重写

每个 train_*.py 在最后会:
1. 选 test R² 最贴近 10 种子中位数的 seed 作 final model
2. 用该 seed 重新 fit 一遍完整训练集 final scaler + final model
3. 写 `*_model.{pkl,h5}` + scaler + `*_seeds_results.json` + `*_model_card.md`

`model_card.md` 内的训练日期 / SHAP top-5 / R² 均值会自动刷新。如果你 retrain 后看到 model card 时间变了但 R² 数字一样,这是 expected。
