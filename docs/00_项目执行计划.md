## 0. 现状盘点

### 0.1 已完成（来自论文成果）

- ✅ 31 省份 × 2011—2023 单源面板数据（403 条，11 维特征）
- ✅ XGBoost-SHAP 基线模型：**R² = 0.6293，RMSE = 0.0051**
- ✅ LSTM 验证模型：**R² = 0.6619，RMSE = 0.0046**
- ✅ Top 3 因子识别：**灌溉率 > 洪涝占比 > 日照时数**
- ✅ 双重验证一致性结论
- ✅ 论文初稿（融合优化版 + 总第五版）

### 0.2 项目相对论文的增量（必须新做的工作）

| 维度 | 论文已有 | 项目需新增 |
|------|---------|-----------|
| 数据 | 统计 + 气象 + 致灾（单模态） | **+ MODIS 遥感 NDVI/LST + GIS 矢量**（多模态融合）|
| 模型 | XGBoost-SHAP + 普通 LSTM | **Attention-LSTM**（注意力机制升级）|
| 系统 | 仅设计构想 | **Vue + Flask + PostgreSQL** Web 系统工程落地 |
| 输出 | 论文 | 系统 + 31 省差异化路径报告 + 竞赛/论文/软著 |

### 0.3 关键风险

1. **数据风险**：MODIS 遥感批量下载和栅格→省级聚合需要技术验证（NASA EARTHDATA + GDAL/rasterio）
2. **时间风险**：12 个月开发 + 4 阶段，第三阶段（系统开发，2.5 个月）压力最大
3. **能力匹配风险**：团队 5 人但部分成员（周煜楠 2025 级）刚入学，需快速培训
4. **目标稳健性风险**：当前 R² ≈ 0.66 已基本达到 ≥ 0.62 目标，但加多模态后可能短期下滑，需调参

- XGBoost 1.7+ / SHAP 0.41+
- TensorFlow 2.x / Keras（或 PyTorch）+ LSTM + Attention 自定义层

### 3.4 风险与应对

- ❗ 加遥感后 R² 可能短期下滑 → 保留单模态对照；逐步加特征

## 7. 协作机制与质量保障

### 7.1 节奏

- **周例会**（线下/线上 60 分钟，每周日晚）
- **每月 last 周**：里程碑同步 + 邱老师 review
- **季度评审**：每阶段结束（M1/M2/M3/M4）

### 7.2 工程规范

- **代码仓库**：GitHub（建议 Private → 开源）
- **分支策略**：main / dev / feat-* / fix-*
- **CI**：GitHub Actions（lint + 单测）
- **文档**：仓库 docs/ + Notion 知识库
- **任务追踪**：GitHub Projects 或飞书任务

### 7.3 数据与代码安全

- 数据备份：本地外置硬盘 + 云对象存储
- 模型版本：MLflow 或简单 `models/{date}_{ver}/` 目录
- 敏感信息：`.env` + `.gitignore`（绝不提交 API key）
