## 6. 后端集成契约

潘会写 `backend/services/inference.py`，**你们只需要保证以下函数签名跑得通**：

```python
# backend/services/inference.py（潘负责实现）
from typing import Protocol, TypedDict

class XGBOutput(TypedDict):
    risk: float                    # 0.005–0.055
    shap_values: dict[str, float]  # 11 个特征的 SHAP 值

class LSTMOutput(TypedDict):
    risk: float
    attention_weights: dict[str, float] | None  # Attention-LSTM 才有

class ModelInferer(Protocol):
    def load_models(self) -> None: ...
    def predict_xgboost(self, features: dict[str, float]) -> XGBOutput: ...
    def predict_lstm(self, features: dict[str, float]) -> LSTMOutput: ...
    def predict_dual(self, features: dict[str, float]) -> dict: ...

# features dict 用训练时的大写键名（Temp, SPEI, ...）
# 内部由 _to_training_keys() 从 API 小写映射过来
```

**算法侧不用写这个文件**——只需要保证你们的 `xgb_model.pkl / lstm_model.h5` 在 Python 里能这样加载并推理：

```python
import joblib, tensorflow as tf, json

xgb = joblib.load("models/xgb_model.pkl")
lstm = tf.keras.models.load_model("models/lstm_model.h5", compile=False)
scaler = joblib.load("models/scaler.pkl")
columns = json.load(open("models/feature_columns.json"))

# 应该能直接跑
features = scaler.transform([[14.2, -0.2, 4.2, 65.4, 1620, 750, 2240, 360, 4.5, 3.8, -0.05]])
print(xgb.predict(features))     # → 单值 risk
print(lstm.predict(features))    # → 单值 risk（注意 LSTM 输入形状可能要 reshape）
```

如果你训练时用了**自定义层**（Attention 必有）、**custom loss / metric**，要么注册 `custom_objects`，要么写一个 `model_loader.py` 一起交付。
