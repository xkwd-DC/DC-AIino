## 6. YAGNI / 可回滚 / 早上线

- **YAGNI**：Phase 1-3 不做用户系统、不做 PostgreSQL（SQLite 够用）、不做 Redis、不做 Docker compose
- **可回滚**：每个 Phase 一个 PR；CI 跑 pytest；merge 后立即在 GCP 部署，出问题立即 `git revert`
- **早上线**：6/25 必须公网可访问，比"完美"重要

> 最后一句：**目标是答辩日评委扫码就进系统，看到一个能用、有数据、有响应的工具。不是写一个完美的工程。**
