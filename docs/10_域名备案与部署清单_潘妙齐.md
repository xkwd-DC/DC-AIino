## 🚨 v1 → v2 路线变更（2026-05-21）

**v1（已废弃）**：阿里云 `.cn` 域名 + 阿里云轻量服务器 + ICP 备案 → 等 20 天 → 6/20 前后可用
**v2（采纳）**：GCP `e2-medium asia-east1` + 国际域名 + 直接 HTTPS → 6/25 前公网可用

**变更动因**：
1. 周煜楠 UI 5/21 提前交付，系统开发提前 7 个月启动（5/24 → 6/25 上线）
2. 备案窗 ~20 天 + 实名认证 ~5 天 + 服务器购买 ≥ 3 个月 → 总投入 4+ 周，与本周冲刺节奏不匹配
3. GCP 学生免费额度 + Google Cloud Free Tier $300 美元 → 比阿里云便宜 + 可立即开通
4. 国内访问延迟 asia-east1 ~80ms，demo / 答辩室可接受
5. 答辩老师扫码即可访问，不卡备案审核

**代价**：
- 国内访问比国内 IDC 慢（80ms vs 20ms），但 demo 不卡顿
- `.cn` 域名不可用，改 `.com / .app / .ai / .io` 国际域名
- 阿里云高校优惠不可用，但 GCP 学生 + 免费额度更便宜

**保留资产**：
- 论文 §5 / 申报书 §5 章节"部署架构"已写阿里云路径，**Phase 5 后回头改写**
- 如答辩要求"国产化部署"，再切阿里云 + 备案（保留 v1 文档作 fallback）

## 2. 服务器实例

### 2.1 配置选择

| 项 | 推荐 | 月费估算（按需）|
|---|---|---|
| 实例类型 | **e2-medium**（2vCPU / 4GB） | ~$25 / 月 |
| 区域 | **asia-east1-a**（台湾彰化）| — |
| OS | **Ubuntu 22.04 LTS** | — |
| 磁盘 | **30GB SSD 永久磁盘** | ~$5 / 月 |
| 防火墙 | 22 (SSH) + 80 (HTTP) + 443 (HTTPS) | — |
| **总计** | | **~$30 / 月** |

**预算来源**：docs/00 § 8 经费"软件与计算资源费 0.50 万"——5 个月 GCP $150 ≈ ¥1100，留 4000 缓冲给数据存储 / 域名 / 答辩临时升配。

### 2.2 创建命令（GCP Console）

```
Compute Engine → 创建实例
├─ 名称：grain-risk-prod
├─ 区域：asia-east1
├─ 区可用区：asia-east1-a
├─ 机器类型：e2-medium
├─ 启动磁盘：Ubuntu 22.04 LTS / 30GB SSD
├─ 防火墙：✅ 允许 HTTP / ✅ 允许 HTTPS
└─ SSH 公钥：粘贴本地 ~/.ssh/id_rsa.pub
```

### 2.3 静态外部 IP（必做）

```
VPC 网络 → IP 地址 → 预留静态外部 IP
├─ 名称：grain-risk-static-ip
├─ 类型：标准
└─ 关联到：grain-risk-prod 实例
```

> ⚠️ 不预留 = 实例重启 IP 会变，域名解析失败。**月费 ~$0**（已使用）。

## 4. 服务器初始化（SSH 后跑）

```bash
# 系统更新
sudo apt update && sudo apt upgrade -y

# 基础工具
sudo apt install -y python3.11 python3.11-venv python3-pip \
  nodejs npm nginx git build-essential certbot python3-certbot-nginx \
  ufw fail2ban

# 升 Node 到 20.x
curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
sudo apt install -y nodejs

# PM2 进程管理
sudo npm install -g pm2

# Claude Code 服务器端
sudo npm install -g @anthropic-ai/claude-code
# 然后跑 `claude` 登录

# 防火墙
sudo ufw allow 22/tcp
sudo ufw allow 80/tcp
sudo ufw allow 443/tcp
sudo ufw enable
```

> ⚠️ Python 装 3.11 不装系统默认（3.10），因为 tensorflow-cpu~=2.17 + numpy<2.1 在 3.11 上 wheel 齐全。

## 6. Nginx 反代

`/etc/nginx/sites-available/grain-risk.conf`：

```nginx
server {
    listen 80;
    server_name grainrisk.app www.grainrisk.app;

    # certbot 申请 HTTPS 时需要这段
    location /.well-known/acme-challenge/ {
        root /var/www/certbot;
    }

    location / {
        return 301 https://$host$request_uri;
    }
}

server {
    listen 443 ssl http2;
    server_name grainrisk.app www.grainrisk.app;

    ssl_certificate     /etc/letsencrypt/live/grainrisk.app/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/grainrisk.app/privkey.pem;

    # 前端静态
    root /opt/grain-risk/frontend/dist;
    index index.html;

    location / {
        try_files $uri $uri/ /index.html;
    }

    # 后端 API
    location /api/ {
        proxy_pass http://127.0.0.1:5000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }

    # gzip / 缓存等优化
    gzip on;
    gzip_types text/css application/javascript application/json;

    location ~* \.(jpg|jpeg|png|gif|ico|css|js|woff2)$ {
        expires 1y;
        add_header Cache-Control "public, immutable";
    }
}
```

激活：

```bash
sudo ln -s /etc/nginx/sites-available/grain-risk.conf /etc/nginx/sites-enabled/
sudo nginx -t  # 校验
sudo systemctl reload nginx
```
