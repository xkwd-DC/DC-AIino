### §3.5 T5 GCP 上线 5 步（详细）

回答用户问 "GCP 需要做什么"：

| 步 | 命令 / 动作 | 预估 | 风险 / 注意 |
|---|---|---|---|
| 1 | GCP console 开 e2-medium 实例（us-west1 或 asia-east2 香港，**香港延迟更低但贵 ~30%**）+ 预留固定外网 IP + 防火墙放 80 / 443 / SSH | 10 min | 学生免费额度需绑卡；e2-medium 月费 ~ $20-25，1 万经费内可控 |
| 2 | SSH 上去：`apt update && apt install -y nginx python3.10 python3.10-venv certbot python3-certbot-nginx git` | 15 min | venv 路径冲突；TF 在 CPU only 实例上推理性能需验证 |
| 3 | `git clone DC` + `python3.10 -m venv .venv && pip install -r backend/requirements.txt` + 跑 `scripts/load_panel_to_db.py`（Phase 4 SQLite ETL）+ `gunicorn -b 127.0.0.1:5000 backend.app:app` | 30-45 min | TF 加载慢（~30s 冷启）；Phase 4 ETL 待 5/27-28 完成 |
| 4 | DNS 把 `grainrisk.app` 加 A 记录指向 GCP 固定 IP（域名服务商控制台，约 5-30 min 生效）→ `certbot --nginx -d grainrisk.app -d www.grainrisk.app` 拿 TLS 证书 | 30 min（含 DNS 传播） | DNS 未生效会让 certbot 失败，需 watch 后重试 |
| 5 | 浏览器访问 `https://grainrisk.app` → 全流程 sanity：首页 / M01 风险地图 / M04 韧性路径 / `/api/predict` POST + `/api/provinces?year=2022` | 15 min | CORS / mixed content / nginx upstream timeout 三大常见 issue |

**总工时**：1 整天（潘 5/27 - 5/28 任选一天完成）。
**部署素材已就位**：`deploy/nginx.conf`、`deploy/setup_letsencrypt.sh`、`backend/requirements.txt`、训练 artifacts 全部在 git 里。
**唯一卡点**：用户需要先把 GCP 账号付费开通（学生免费额度即可，但需绑卡激活）。
