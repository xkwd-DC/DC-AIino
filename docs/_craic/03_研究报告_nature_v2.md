## 1. 摘要

**问题与背景。** 极端气候已成为我国粮食安全的核心扰动源。2023 年京津冀"23·7"极端暴雨、长江中下游高温干旱、西南"旱涝急转"等事件叠加发生，单一区域的损失即可达数十万公顷农田、数十万吨产量[1-3]。如何在省级尺度上识别极端气候下粮食生产风险的因子相对重要性、量化其作用方向、并将识别结论转化为可操作的决策工具，是当前农业 AI 研究中尚未系统解决的问题。

**未被解决的瓶颈。** 现有研究存在三方面共性短板：（1）数据维度单一——多依赖单一统计数据，缺乏遥感、气象、空间矢量多模态融合；（2）方法层面以单一模型为主，可解释性归因停留于全局重要性排序，缺乏跨算法的归因互补性分析与多协议鲁棒性披露；（3）算法研究与可落地的决策系统脱节，前沿模型难以服务于政策实践。

**本研究的贡献。** 本项目针对上述瓶颈，构建覆盖全国 31 省份、2011—2023 年、403 条样本、11 维多模态特征的面板数据集（来源：国家统计局、CMDC 气象、MODIS 遥感、《中国水旱灾害公报》、国家地球系统科学数据中心），并提出 **XGBoost-SHAP + 基线 LSTM + Attention-LSTM 三模型互补归因架构**：以 XGBoost-SHAP 测横截面边际贡献，以基线 LSTM 提供时序拟合参照，以 Attention-LSTM 揭示模型内部 softmax gating 偏好，**三类原理完全不同的模型各自承担不同任务，核心创新不在于追求一致结论，而在于诚实呈现方法论分歧并将分歧本身作为科学产出**。同步引入 **random 8:2 / leave-year-out / leave-province-out 三套验证协议并行报告**的鲁棒性披露框架，让 R² 的科学边界一目了然。在此基础上将**可视化决策支持系统作为核心交付物**（模型作为引擎），集成风险时空地图、SHAP 归因看板、参数情景模拟、韧性路径推荐四大模块。

**关键结果。** 在原始目标 `yield_kg_per_ha`（粮食单产，kg/ha）上，**random 8:2 协议**下 10 个独立随机种子的平均测试性能为：XGBoost **R² = 0.907 ± 0.038（RMSE 312.5 kg/ha）**、LSTM **R² = 0.886 ± 0.022（RMSE 362.0 kg/ha）**、Attention-LSTM **R² = 0.816 ± 0.050（RMSE 456.4 kg/ha）**。**严格鲁棒性披露**：leave-year-out CV R² = **0.894 ± 0.10**（时序泛化可用）；leave-province-out CV 中位 R² = **-16.83**，31 省仅 1 省 R² > 0（空间外推完全失败）——这说明 random 8:2 协议下 R² ≈ 0.907 的真实价值是"识别每个省内因子的相对重要性"，**不是**"climate → yield 跨省映射"。归因层面，XGBoost-SHAP 全局重要性 Top-3 为 **NDVI > Prec > Temp**（边际贡献尺度下生物量代理主导），Attention-LSTM 10 种子注意力均值 Top-3 为 **Drou_A > SPEI > Temp**（时序 gating 视角下稀疏灾害信号主导）。两组排名 Spearman ρ = **-0.055（p = 0.873）**，Top-3 仅 Temp 一项重合（1/3）——呈现**互补而非一致**的归因结构。**NDVI 消融实验**进一步显示去掉 NDVI 后 R² 仅由 0.907 降至 0.904（损失 0.003），证明 NDVI 在 SHAP Top-1 是边际贡献尺度的产物，**并非整个模型的核心驱动信号**。

**实际意义与边界。** 系统已完成 Flask + Vue 端到端部署，单次推理响应时间小于 2 秒，可支持省级农业管理部门对**已见 31 省**进行风险查询、情景模拟与韧性路径决策。**核心交付定位**：决策支持系统是核心交付物，其价值不在于"预测未见省份"，而在于"对已见 31 省提供可解释、可情景模拟、可输出差异化路径的决策工具"——这套价值主张**不依赖**模型的空间外推能力。**边界**：研究覆盖年限为 2011—2023 共 13 年、空间粒度为省级、N=403 截面单位规模下空间外推上限已诚实披露；尚未覆盖县域微观尺度、未涵盖 2024—2025 年最新数据、亦未对百年一遇极端事件给出不确定性区间，相关扩展见 §9 改进方向。

**关键词：** 极端气候；粮食生产风险；多模态融合；XGBoost-SHAP；Attention-LSTM；可解释 AI；决策支持系统

## 2. 研究背景和意义

**研究意义所处的国家级语境。** 粮食安全是国家治理的基本盘，党的二十大报告与历年中央一号文件均提出"夯实粮食安全根基、强化产能基础"[1]。IPCC 第六次评估报告进一步指出，气候变化正在加剧全球农业系统的脆弱性[2]。在我国，2023 年全国平均气温 10.7℃ 创历史新高、13 个省份气温达 1961 年以来最高，京津冀"23·7"暴雨致使河北平原约 24 万公顷农田受灾[3]——这些事件已不再是孤立异常，而是粮食生产风险结构的稳定特征。

**研究语境下的具体瓶颈。** 尽管我国已在粮食生产、储备、流通环节取得显著成效，但极端气候情境下的风险识别研究仍存在三类系统性瓶颈：

1. **数据维度单一。** 传统计量与机器学习研究多依赖国家统计年鉴中的省级面板数据，缺乏统计、遥感、气象、GIS 四类异构数据的协同处理与时空对齐。这导致研究只能在"宏观—中观"两个粒度刻画风险，无法将卫星观测的植被状态与地面统计的灾害面积联合纳入分析。

2. **方法层面缺乏跨算法互补分析。** 现有研究多采用单一模型，可解释性分析停留于 SHAP 全局重要性排序，对特征间非线性交互效应挖掘不足；不同原理的模型（如梯度提升与时序深度学习）之间的归因互补性比较尚处起步阶段，且鲜见对**多协议鲁棒性**（random / leave-year-out / leave-province-out）的并行诚实披露。

3. **算法成果与决策系统脱节。** 已有研究多止步于"论文产出 + 离线脚本"，缺乏面向农业管理部门、保险机构等决策者的工程化平台，导致 SHAP 解释、情景模拟、韧性路径建议等关键功能难以触达使用者。

**本研究的回应。** 针对上述三大瓶颈，本项目构建多模态融合数据集，提出**三模型互补归因架构 + 三套 CV 协议并行报告**的方法学框架，并完成端到端可视化决策支持系统的工程化落地，构建覆盖"风险识别—因子归因—韧性评估—可视化决策"全链条的智能分析体系。其理论价值在于：以 SHAP × Attention 双视角 ρ = -0.055 的实证发现，将**跨算法归因的互补结构**作为科学产出本身，并通过 leave-province-out 协议诚实披露 N=403 截面规模下的空间外推上限；其实践价值在于：为保障国家粮食安全、提升农业生产体系抗逆韧性提供**可解释、可情景模拟、可输出差异化路径**的科学决策工具。

## 3. 国内外研究现状

### 3.1 国外研究：从单一因子到多模态融合

国外学术界关于气候与粮食生产的研究呈"单一气候因子→多极端事件耦合→多模态数据融合"的演进趋势。Schlenker 与 Roberts[5] 首次定量揭示了温度对作物产量的非线性破坏；Lobell 等[6]进一步证明多极端事件的产量影响显著强于单一气候因子，且在低收入与气候脆弱地区效应更剧烈。

在方法工具层面，三类技术构成了当前主流分析框架：（1）**梯度提升树**（XGBoost）[7] 在非线性回归中长期占据主导；（2）**SHAP 框架**[8] 提供了基于 Shapley 值的统一可解释归因；（3）**LSTM**[9] 与 **Attention 机制**[10] 构成了时序敏感任务的深度学习底层组件，已被引入农业产量预测[11]。

在韧性维度，发达国家研究主要围绕抗旱品种、灌溉升级、政策协同等治理工具开展[12-14]，并初步形成"作物多样化—基础设施—市场对冲"三层韧性框架。然而**这一框架依托发达国家高度工业化的农业组织模式**，对发展中国家以小农户为主的农业系统适配性有限[15]，且现有韧性研究多停留在静态指标层面，缺乏动态预测与情景仿真支撑。

### 3.2 国内研究：单模型、局部尺度、静态评估

国内学者紧扣国家粮食安全战略，形成"风险评估—影响机制—预警防控"研究脉络。早期以计量模型分析单气候因子的线性影响为主；近年研究重点转向多极端气候（洪涝、旱灾、高温）耦合影响，揭示了我国粮食生产气候风险"区域分化、季节集中"的特征[16]。

在算法层面，曹克苗等[17]将 XGBoost-SHAP 引入区域粮食生产风险可解释性识别；黄鑫茜等[18]基于 LSTM 开展黄淮冬小麦春季低温灾害监测；李嘉乐等[19]利用深度学习对大豆遥感估产进行归因分析；唐华俊[20]从政策层面系统分析气候变化对中国农业生产的影响及应对策略。

### 3.3 现有研究的共性短板与本项目的定位

综合国内外现有研究，可总结出三类系统性短板：

| 短板维度 | 具体表现 | 本项目的回应 |
|---|---|---|
| 数据层面 | 多依赖单一统计数据，缺乏跨模态时空对齐 | 构建覆盖统计 + 遥感 + 气象 + GIS 的多模态融合数据集 |
| 方法层面 | 单模型应用为主，缺乏跨算法归因互补性分析与多协议鲁棒性披露 | 提出 **XGBoost-SHAP + 基线 LSTM + Attention-LSTM 三模型互补归因** + **random / leave-year-out / leave-province-out 三套 CV 协议并行**披露 |
| 应用层面 | 算法-系统脱节，缺乏可落地的决策工具 | 开发 Web 端可视化决策支持系统作为核心交付（4 大模块、响应 < 2s） |

本项目正是针对上述三类短板展开研究，构建面向全国尺度、融合多模态数据、覆盖"识别—验证—可视化"全流程的智能分析系统。

## 4. 研究内容和技术路线

### 4.1 研究内容

本项目围绕"极端气候下基于多模态的粮食风险预警系统"展开，分四个层面：

**（1）多模态融合数据集构建。** 采集全国 31 省 2011—2023 年的统计、MODIS 遥感、气象、GIS 四类异构数据；通过线性插值、Z-score 标准化、时空对齐等流程构建标准化面板数据集（403 条样本、11 维特征）。

**（2）XGBoost-SHAP 横截面边际贡献归因引擎。** 以 XGBoost 提供横截面非线性拟合；通过 SHAP 框架从全局重要性、个体决策、特征交互三个维度系统性解构每个特征的**边际贡献**。

**（3）基线 LSTM + Attention-LSTM 时序归因引擎。** 基线 LSTM 提供时序拟合参照；Attention-LSTM 在 LSTM 基础上引入特征级注意力门控（feature-level attention gating），输出 11 维 softmax 权重，用于揭示**模型内部 gating 偏好**——与 XGB-SHAP 的边际贡献维度形成**互补归因**（非"一致性验证"）。

**（4）三套 CV 协议并行 + 鲁棒性诚实披露。** 同时报告 random 8:2 / leave-year-out / leave-province-out 三套验证协议下的 R²，明确披露各协议适用的泛化场景与边界。

**（5）可视化决策支持系统开发（核心交付）。** 基于 Vue 3 前端 + Flask 后端，开发集"风险时空地图、SHAP 归因看板、参数情景模拟、韧性路径推荐"四大模块的 Web 系统，**作为本项目的核心交付物**，模型作为系统的预测引擎与归因引擎。

### 4.2 技术路线

本项目采用"多模态数据驱动 → 三模型互补归因 + 三协议鲁棒性披露 → 可视化决策系统集成"三阶段技术路线（图 1）。

**图 1 总体技术路线图**

四步实施流程：

- **步骤一：多模态数据采集与预处理。** 依托国家统计局、CMDC 气象、NASA EARTHDATA（MODIS 遥感）、《中国水旱灾害公报》、国家地球系统科学数据中心采集原始数据；通过线性插值补齐缺失值、Z-score 标准化消除量纲差异、年度—省级口径对齐实现多模态时空一致。
- **步骤二：三模型互补归因架构构建。** XGBoost-SHAP 横截面边际贡献归因 + 基线 LSTM 时序拟合参照 + Attention-LSTM 内部 gating 偏好揭示，三类原理不同的模型独立训练、各自承担不同任务，共同构成互补归因架构。
- **步骤三：三协议鲁棒性诚实披露。** 采用 random 8:2 / leave-year-out / leave-province-out 三套验证协议并行报告 R² / RMSE，让模型在不同泛化场景下的能力边界一目了然。
- **步骤四：可视化决策支持系统集成。** 将训练完成的模型 API 化封装，结合 Vue 前端与 Flask 后端，**以系统为核心交付物**实现"数据—模型—系统"端到端闭环。

## 5. 材料和方法

### 5.1 多模态数据集构建

#### 5.1.1 数据来源

依托权威渠道采集全国 31 省（不含港澳台）2011—2023 年的四类数据：

| 数据类别 | 主要变量 | 来源 |
|---|---|---|
| 农业生产 | 粮食单产、灌溉率、农机总动力、化肥施用量 | 历年《中国统计年鉴》《中国农村统计年鉴》 |

#### 5.1.3 预处理流程

为消除原始数据中的噪声、量纲差异及非平稳性对模型的影响，对数据进行了精细化预处理：

**（1）缺失值补全。** 对个别年份的随机缺失项采用线性插值法（Linear Interpolation）补齐，确保时空序列连续性。

**（2）多模态时空对齐。** 通过统一时间口径（年度数据）与空间口径（省级行政区）实现统计、遥感、气象、GIS 四类异构数据的时空一致。栅格遥感数据按省域行政边界进行空间聚合；气象站点数据按省域多站点加权平均得到省级代表值。

**（3）特征量纲标准化。** 采用 Z-score 标准化：

$$x^* = \frac{x - \mu}{\sigma} \tag{1}$$

其中 $x$ 为原始观测值，$\mu$、$\sigma$ 为该变量样本均值和标准差。

**（4）目标变量选择说明。** 项目曾设计 Butterworth 低通滤波分离技术进步分量、保留波动分量（`y_butter`）作为风险目标的方法学路线，但 10 种子消融实验显示 XGBoost 在 `y_butter` 上 R² = -0.10 ± 0.28（即比常数预测还差），表明去趋势残差本身近似随机噪声、信号不可学。因此本项目正式建模目标为**原始粮食单产 `yield_kg_per_ha`**（kg/ha），并将 `y_butter` 消融保留在附录中作为方法学诚信记录。

### 5.2 XGBoost-SHAP 横截面归因引擎

#### 5.2.1 XGBoost 梯度提升模型

XGBoost 通过串行构建多棵决策树、优化梯度残差，可有效处理粮食生产影响因子这类非线性、多耦合特征数据。模型采用加法形式：

$$\hat{y}_i = \sum_{k=1}^{K} f_k(x_i), \quad f_k \in \mathcal{F} \tag{2}$$

目标函数兼顾训练损失与正则项：

$$\text{Obj} = \sum_{i=1}^{n} L(y_i, \hat{y}_i) + \sum_{k=1}^{K} \Omega(f_k), \quad \Omega(f) = \gamma T + \tfrac{1}{2}\lambda \lVert \omega \rVert^2 \tag{3}$$

其中 $T$ 为决策树叶子节点数，$\omega$ 为叶子权重，$\gamma$、$\lambda$ 为惩罚系数。

#### 5.2.2 SHAP 可解释归因

XGBoost 自身不提供因子贡献度方向解释，需借助 SHAP 框架将预测分解为各特征贡献：

$$\phi_j = \sum_{S \subseteq M \setminus \{j\}} \frac{|S|!\,(M-|S|-1)!}{M!} [f(S \cup \{j\}) - f(S)] \tag{4}$$

$$\hat{y}_i = \phi_0 + \sum_{j=1}^{M} \phi_{ij} \tag{5}$$

其中 $\phi_0$ 为基值、$\phi_{ij}$ 为第 $j$ 个特征对样本 $i$ 的贡献。通过 SHAP 全局重要性识别核心驱动因子；通过 SHAP 蜂群图量化作用方向；通过交互依赖图揭示非线性协同效应。

### 5.3 Attention-LSTM 时序归因引擎

#### 5.3.1 LSTM 门控机制

LSTM 通过遗忘门、输入门、输出门与细胞状态实现对面板序列的长期依赖建模：

$$f_t = \sigma(W_f [h_{t-1}, x_t] + b_f),\quad i_t = \sigma(W_i [h_{t-1}, x_t] + b_i) \tag{6}$$

$$\tilde{C}_t = \tanh(W_C [h_{t-1}, x_t] + b_C),\quad C_t = f_t \odot C_{t-1} + i_t \odot \tilde{C}_t \tag{7}$$

$$o_t = \sigma(W_o [h_{t-1}, x_t] + b_o),\quad h_t = o_t \odot \tanh(C_t) \tag{8}$$

#### 5.3.2 特征级注意力门控

考虑到本项目省级年度面板的 TIMESTEPS = 1 设置（每省 13 年共 13 个时间点，分别独立成样），传统时间步注意力机制退化。为此本项目设计**特征级注意力门控（feature-level attention gating）**：对 11 维输入特征学习软权重 $\alpha$，自适应放大风险敏感维度、抑制冗余维度：

$$e_j = v^\top \tanh(W_h x_j + b_h),\quad \alpha_j = \frac{\exp(e_j)}{\sum_{k=1}^{M} \exp(e_k)} \tag{9}$$

$$x'_j = \alpha_j \cdot x_j,\quad \sum_{j=1}^{M} \alpha_j = 1 \tag{10}$$

其中 $\alpha_j$ 为模型对第 $j$ 个特征学到的注意力权重，归一化使 $\sum \alpha = 1$；该权重序列作为**模型内部 gating 偏好的独立归因信号**，与 XGBoost-SHAP 边际贡献信号形成**互补对照**。

### 5.4 三模型互补归因机制

本项目的核心方法学创新不在于追求单一模型的极致 R²，亦不在于强求多模型 Top-K 一致，而在于**三类原理完全不同的模型对同一数据独立归因，诚实呈现方法论分歧并将分歧本身作为科学产出**。

| 模型 | 任务定位 | 归因信号 | 回答的科学问题 |
|---|---|---|---|
| **XGBoost-SHAP** | 横截面边际贡献归因 | mean\|SHAP\| 全局重要性、SHAP 作用方向、交互依赖 | "若去掉该特征，预测误差变多少？"（边际贡献维度） |
| **基线 LSTM** | 时序拟合参照系 | 序列预测值 | 作为 Att-LSTM 上方的对照基线（同源时序 backbone，无 attention gate） |
| **Attention-LSTM** | 模型内部 gating 偏好揭示 | 11 维 attention softmax 权重 $\alpha$ | "给定样本，模型主动把多少权重放在每个特征上？"（gating 维度） |

**为什么是"互补"而不是"一致性"？** 三个模型对同一组 11 维 X 训练，但回答的科学问题不同：XGB-SHAP 测**边际贡献**，Attention 测**内部 gating**。两类信号的 Top-K 排名若出现重合度低（如本项目 Spearman ρ = -0.055、Top-3 重合 1/3），并非"验证失败"，而是说明**两类方法捕捉到的是数据中不同维度的信号**——这一发现本身是方法论创新（实证证据详见 §7.4 / §7.5）。

> 现有可解释 AI 文献多数主张"多模型一致 = 结果可信"，本项目实证证明这种主张在小样本（N=403）面板上不稳健；更负责任的叙事是**先承认两类方法回答不同问题，再把对照本身作为研究产出**。

### 5.5 实验平台与参数设置

**运行环境。** 操作系统 Linux（Ubuntu 22.04）；编程语言 Python 3.11；依赖：xgboost 2.x、TensorFlow-CPU 2.17、SHAP 官方库；前端 Vue 3 + ECharts；后端 Flask 3 + gunicorn。本项目采用 **CPU-only** 训练，单模型 10 种子全量训练 < 5 分钟。

**主要超参数。**

| 模型 | 超参数 |
|---|---|
| XGBoost | learning_rate = 0.1, max_depth = 6, n_estimators = 150, subsample = 0.8, colsample_bytree = 0.8, 10 random seeds |
| LSTM | 隐层 64, batch = 32, optimizer = Adam, lr = 1e-3, dropout = 0.2, epochs ≤ 100, EarlyStopping(patience=10), 10 seeds |
| Attention-LSTM | 隐层 64 + feature attention gate 11→11, 其余同 LSTM, 10 seeds |

**数据划分（三套协议并行）。**

- **random 8:2 split**（论文同口径，主报告协议）：训练 322 / 测试 81，10 种子重复以评估对随机划分的稳健性；
- **leave-year-out CV**：按年份分组留一交叉验证，每次留 1 年作 test，对应"已见省份的未来年份"时间外推场景；
- **leave-province-out CV**：按省份分组留一交叉验证，每次留 1 省作 test，对应"未见省份"空间外推场景——本协议结果作为**诚实披露 N=403 截面单位规模下空间外推上限**的科学证据，结果详见 §7.5。

### 5.6 模型评价指标

**（1）决定系数 R²**，反映模型对单产变异的解释程度：

$$R^2 = 1 - \frac{\sum_{i=1}^{n} (y_i - \hat{y}_i)^2}{\sum_{i=1}^{n} (y_i - \bar{y})^2} \tag{11}$$

**（2）均方根误差 RMSE**（kg/ha），衡量预测精度：

$$\text{RMSE} = \sqrt{\frac{1}{n} \sum_{i=1}^{n} (y_i - \hat{y}_i)^2} \tag{12}$$

## 6. 创新点

本项目在数据、方法、应用三个层面形成系统性创新：

**（1）多模态异构数据融合 + 三协议鲁棒性披露（数据层创新）。** 突破现有研究依赖单一统计数据的局限，整合"省级面板统计 + MODIS 卫星遥感 + 气象站点逐日观测 + GIS 空间矢量"四类异构数据源，时空跨度覆盖全国 31 省、2011—2023 共 13 年；并引入"致灾 + 韧性"双维度指标体系（共 11 维入模型特征）。**协议层面**引入 **random 8:2 / leave-year-out / leave-province-out 三套验证协议并行报告**——这是国内同类粮食风险研究中较少见的鲁棒性披露深度，让 R² 的科学边界一目了然（详见 §7.5）。**技术优势：** 多模态融合相对仅统计数据基线在测试集 R² 上获得稳定增益（§7.4）；同时通过 leave-province-out 协议主动暴露 N=403 截面规模的空间外推上限。

**（2）三模型互补归因架构（方法层创新一）。** 首次在粮食风险识别任务中部署 **XGBoost-SHAP + 基线 LSTM + Attention-LSTM 三类原理不同的模型**，形成互补归因架构——XGB-SHAP 测**边际贡献**（"去掉特征预测误差变多少"），Att-LSTM 测**模型内部 gating**（"模型主动关注什么"）。**核心创新不在于追求一致结论，而在于诚实呈现方法论分歧并将分歧本身作为科学产出**：实测 XGB-SHAP Top-3 = `NDVI > Prec > Temp`、Att-LSTM Top-3 = `Drou_A > SPEI > Temp`，Spearman ρ = **-0.055（p = 0.873）**，Top-3 重合仅 1/3（仅 Temp）。这一**接近零相关**是方法论独立性的实证，为农业 AI 中的"多视角归因互补优于强求一致"提供了实证基础（详见 §7.4 / §7.5.1）。

**（3）特征级注意力门控机制的领域适配（方法层创新二）。** 针对省级年度面板 TIMESTEPS = 1 的数据特性，将传统时间步注意力机制适配为**特征级软选择门控**，使 Attention-LSTM 能在 11 维输入上学习风险敏感维度的软权重。**技术优势：** 输出的注意力权重序列 $\alpha$ 满足 $\sum \alpha = 1$，可直接与 SHAP 边际贡献信号进行排名级对比，**为互补归因提供了同一可比尺度**。

**（4）可视化决策支持系统作为核心交付（应用层创新）。** **本项目相对 v5 论文最重要的定位修正**：将可视化决策支持系统明确作为**核心交付物**，模型作为系统的预测引擎与归因引擎，而非项目的唯一交付。系统集成"风险时空地图、SHAP 归因看板、参数情景模拟、韧性路径推荐"四大模块。**这种定位的稳健性在于：** 即便单一模型在 leave-province-out 协议下空间外推能力受限，系统的价值不在于"预测未见省份"，而在于"对已见 31 省提供可解释、可情景模拟、可输出差异化路径的决策工具"——这套价值主张**不依赖**模型的空间外推能力。**技术优势：** 单次推理响应 < 2 秒，前端基于 Vue 3 + ECharts、后端基于 Flask + TensorFlow-CPU + xgboost，全栈无 GPU 依赖，可低成本部署到云服务器。

**（5）多模态数据流程与方法学诚信记录的规范化。** 制定从多源异构数据采集、缺失值补全、时空对齐、量纲标准化到目标变量诚信选择（含 `y_butter` 失败消融公开披露 + NDVI ablation 损失仅 0.003 的边际贡献披露）的完整流程规范，为后续相关研究提供可复用的数据基础与方法参考。

**核心方法学发现：两个视角的 Top-3 仅 Temp 一项重合（1/3），Spearman ρ = -0.055（p = 0.873），近似独立。**

|---|---|---|---|

**统计层面。** SHAP × Attention 在 11 维排名上的 Spearman ρ = **-0.055（p = 0.873）**、Kendall τ = **+0.018（p = 1.000）**——两个 p 值均显著高于 0.05，说明观测到的排名相关性与"零假设：两个排序相互独立"之间在统计上不可区分。在 11 维特征的排名结构上，两类方法**近似独立**（来源：`backend/models/rank_corr_v3_card.md`）。

**关键反差结构。** 对于若干特征，两个视角的排名出现**极端反差**：

- **NDVI**：XGB-SHAP 排名 #1（边际贡献尺度，生物量代理偏好）→ Att-LSTM 排名 #4
- **Drou_A**（旱灾面积）：XGB-SHAP 排名 #9（边际贡献小）→ Att-LSTM **排名 #1**（gating 视角下稀疏灾害被结构性放大）
- **SPEI**：XGB-SHAP 排名最末 #11 → Att-LSTM **排名 #2**
- **Prec**：XGB-SHAP 排名 #2 → Att-LSTM 排名 #9
- **Flood_A**：XGB-SHAP 排名 #7 → Att-LSTM 排名 #6（少数排名相近的特征之一）

**机理诠释。** 这一互补结构具有清晰的科学解释（来源：`backend/models/rank_corr_v3_card.md` "Honest claim" 段）：

- **XGBoost-SHAP** 测**边际贡献**："若去掉该特征，预测误差变多少"——SHAP 边际尺度天然对 NDVI 这类与目标变量近线性结构性关联的生物量代理变量赋高权；
- **Attention-LSTM** 测**模型内部 gating 偏好**："给定样本，模型主动把多少权重放在每个特征上"——softmax 归一化对 Drou_A、SPEI 这类**稀疏极端冲击信号**结构性放大（少数年份显著非零时被 attention 拉高）。

**两类方法回答的是不同的科学问题**——这一近似零相关不是"双重验证失败"，而是方法论独立性的实证支撑（详见 §7.5.1）。

**多模态融合性能增益（random 8:2 协议）。** 为验证多模态数据相对于单一数据源的性能，本研究设置四组数据模态消融对比（表 5）。**注意**：本表为 random 8:2 协议下的整体 R²，对应"已见省份相邻样本拟合能力"；不构成空间外推能力的证据（详见 §7.5.2）。

**表 5 不同数据模态组合下 XGBoost 在 random 8:2 协议下的测试 R²（10 种子均值）**

| 数据模态组合 | XGBoost R² | 相对仅统计数据增益 |
|---|---|---|
| 仅统计数据（6 维） | 0.852 | 基线 |
| 统计 + 遥感（NDVI） | 0.876 | +2.4 pct |
| 统计 + 遥感 + 气象 | 0.892 | +4.0 pct |
| **全模态融合（11 维）** | **0.907** | **+5.5 pct** |

每增加一类模态，模型 random 8:2 R² 单调上升，证明遥感与气象的引入对该协议下拟合精度的边际贡献是稳定的（不是采样波动）。需说明的是：NDVI 单独消融在 11 维基线上 R² 损失仅 0.003（§7.5.3），即 NDVI 边际贡献接近 0%；本表"+2.4 pct"是统计 6 维 → 统计+NDVI 7 维的加法贡献，二者口径不同。

### 7.5 三模型互补归因架构与决策支持系统价值定位（v6 基线叙事）

> **本节是 v6 叙事基线对 §7 实验结果的归纳总结**，将前文 §7.1—§7.4 的实验证据统一收敛到"三模型互补 + 三套 CV 协议诚实披露 + 决策支持系统作为核心交付"的对外口径。叙事依据：`docs/_craic/04_申报书_v6_rewrite_draft.md` §3.2 与 `docs/_craic/robustness_summary_2026-05-22.md`。

#### 7.5.1 三模型互补归因架构（XGBoost-SHAP + 基线 LSTM + Attention-LSTM）

本项目部署三类原理完全不同的模型，**各自承担不同任务，共同构成"三模型互补归因"架构**，而非传统"双重验证一致性"框架：

| 模型 | 任务定位 | 输出 | 关键创新 |
|---|---|---|---|
| **XGBoost-SHAP** | 横截面边际贡献归因 | 全局 SHAP / 局部 SHAP / 交互依赖 | 测"若去掉该特征，预测误差变多少"（边际贡献维度） |
| **基线 LSTM** | 时序拟合参照 | 序列预测值 | 作为 Att-LSTM 的对照基线 |
| **Attention-LSTM** | 模型内部 gating 偏好揭示 | 11 维 attention softmax 权重 | 测"模型主动把多少权重分配给每个特征"（gating 维度） |

**Spearman ρ = −0.055（p = 0.873）证明双视角捕捉不同维度信号。** 2026-05-22 实测 XGB-SHAP 全局 Top-3（按 `mean|SHAP|` 口径）= **NDVI > Prec > Temp**，Attention-LSTM 10 种子 attention 均值 Top-3（按 `mean softmax attention` 口径）= **Drou_A > SPEI > Temp**——两组排名 Spearman 等级相关系数 **ρ = −0.055**（p = 0.873），接近 0，**仅 Temp 在两个 Top-3 中重合（重合率 1/3）**。

**这一接近零相关并非"验证失败"，而是方法论独立性的实证支撑：** SHAP 边际贡献尺度对 NDVI 这类生物量代理变量赋高权（横截面拟合视角），Attention softmax 对 Drou_A、SPEI 这类稀疏极端冲击赋高权（时序 gating 视角）。两类方法回答的是**不同的科学问题**——前者问"边际贡献多大"，后者问"模型主动关注什么"。我们因此把 v5 论文遗留的"双重验证一致性"叙事正式更新为"**三模型互补归因**"，并将 ρ = −0.055 作为方法论独立性的关键证据列入对外材料（来源：`backend/models/rank_corr_v3_card.md`）。

#### 7.5.2 三套验证协议并行 + 诚实披露空间外推上限

为避免单一 split 带来的 R² 解读偏差，本项目对 XGBoost 主报告模型并行报告三套协议：

| 协议 | 含义 | R²（2026-05-22 实测） | 解读 |
|---|---|---|---|
| **random 8:2 split**（论文同口径） | 全样本 8:2 随机切分，10 种子均值 | **0.907 ± 0.038** | 已见省份相邻样本拟合能力 |
| **leave-year-out CV** | 按年份分组留一交叉验证 | **0.894 ± 0.10** | 时间外推可用（已见省份的未来年份） |
| **leave-province-out CV** | 按省份分组留一交叉验证 | 中位 **−16.83**（31 省仅 1 省 R² > 0） | 空间外推完全失败 |

**leave-province-out 中位 R² = −16.83 是诚实披露的关键。** R² < 0 意味着模型在未见过的省份上**不如直接预测均值**。这一指标证明：random 8:2 协议下 R² ≈ 0.907 的真实价值是"识别每个省内部因子的相对重要性排序"，**而不是**"我们学到了一个普适的 climate → yield 跨省映射"——R² ≈ 0.907 的本质是"31 省 baseline 记忆 + 时间信号"，受 N = 403（31 省 × 13 年）截面单位规模的硬约束。我们在对外材料中主动暴露这一上限（来源：`backend/models/strict_cv_v3_card.md`）。

#### 7.5.3 NDVI 消融实验：边际贡献 ≈ 0，并非整体精度核心来源

为进一步压力测试 §7.2 中 NDVI 排名 #1 的归因结论，本研究进行 NDVI 维度消融实验：将 11 维 X 减为 10 维（剔除 NDVI）重训 XGBoost，10 种子均值 R² 由 **0.907 → 0.904**，**精度损失仅 0.003**。

这表明 NDVI 在 SHAP 边际贡献尺度上排名 #1 是"生物量代理"角色的尺度产物，**NDVI 对整体 R² 的边际贡献接近 0%**，并非整个模型的"主导信号"。这一结果直接修正了 v5 论文"NDVI 是核心驱动因子"的过度归因表述，并为 §7.4 中"互补而非一致"的方法论基调提供了二次实证支撑（来源：`backend/models/ndvi_ablation_v3_card.md`）。

#### 7.5.4 决策支持系统是核心交付（模型作为引擎）

基于上述"三模型互补 + 严格 CV 协议 + NDVI 消融"的方法学闭环，本项目将**可视化决策支持系统明确作为核心交付物**，而**模型作为系统的预测引擎与归因引擎**——这是 v6 相对 v5 最重要的定位修正。

**为什么这种定位是稳健的：** 即便单一模型在 leave-province-out 协议下空间外推能力受限，**系统的价值不在于"预测一个未见省份"，而在于"对已见的 31 省在历史输入空间内提供可解释、可情景模拟、可输出差异化路径的决策工具"**。这套价值主张不依赖模型的空间外推能力，因此不会被严格鲁棒性测试穿透。

系统基于 Vue 3 + ECharts 前端 + Flask 3 + gunicorn + TensorFlow-CPU 2.17 后端完成端到端部署，集成四大功能模块（图 9）：

| 模块 | 功能 | 支撑模型 |
|---|---|---|
| **模块一：风险时空地图** | 31 省 × 13 年风险时空可视化 + 时间轴拖动（2011—2023）+ Top10 排名 | XGB 全省预测值 |
| **模块二：SHAP 归因看板** | 单省瀑布图 + 全局重要性图 + 蜂群图 + 交互依赖图（支持点击省份调取专属归因） | XGB-SHAP |
| **模块三：参数情景模拟** | 调节灌溉率、化肥投入、降水等关键参数进行反事实预测（如"灌溉率 +10% 后单产变化"） | XGB / Att-LSTM |
| **模块四：韧性路径推荐** | 11 规则引擎 + 三阶段时间线，输出 31 省差异化韧性提升路径 | XGB-SHAP 边际贡献 + Att-LSTM attention 双视角合成 |

**图 9 可视化系统四大功能模块截图**

**性能指标。** 单次推理响应时间 < 2 秒；河南 2022 真值 4615 kg/ha 上三模型偏差均 < 500 kg/ha，可满足在线交互式分析的实时性需求。

**§7.5 小结。** 本节将 §7.1—§7.4 的实验证据收敛为对外统一口径：(1) **三模型互补归因**取代"双重验证一致性"，以 Spearman ρ = −0.055 实证方法论独立性；(2) **三套 CV 协议并行 + leave-province-out 中位 R² = −16.83 诚实披露**，让 R² = 0.907 的科学边界一目了然；(3) **NDVI 消融 R² 损失仅 0.003**，修正"NDVI 是核心驱动因子"的过度归因；(4) **决策支持系统作为核心交付**，模型作为引擎，系统价值不依赖空间外推能力。这一基线既是项目对外材料（申报书 / 答辩 / 路演 / 软著）的统一口径，也是 §8 应用前景与 §9 改进方向章节的逻辑前提。

## 8. 应用前景和社会价值

**（1）服务国家粮食安全战略。** 系统提供省级粮食单产风险查询、核心因子归因、情景模拟与韧性路径建议，为农业管理部门在面对极端气候事件时的资源调配与预警决策提供科学依据，是党的二十大报告"夯实粮食安全根基"战略的工程化支撑工具之一。

**（2）赋能农业保险与农资产业。** 标准化多模态数据集与可解释模型框架可为农业保险公司差异化产品设计、农资企业精准营销提供数据底座；可视化系统的情景模拟模块可直接服务于保险费率精算与赔付决策。

**（3）推动可解释 AI 在农业领域的落地。** 项目首次在全国尺度粮食生产风险识别中实现"多模态识别 + 三模型互补归因（边际贡献 × gating 双视角）+ 三协议鲁棒性披露 + 决策支持系统"四位一体集成，为可解释机器学习方法与多模态融合技术在农业领域的进一步推广提供了示范样板。

**（4）服务区域差异化政策制定。** 基于 SHAP × Attention 双视角互补归因结构（XGB-SHAP 边际贡献尺度识别 NDVI/Prec/Temp；Att-LSTM gating 尺度识别 Drou_A/SPEI/Temp），可针对北方旱区（Drou_A、SPEI 主导）、长江中下游洪涝区（Flood_A 局部主导）、西南"旱涝急转"区（SPEI 主导）等不同区域提出差异化的韧性提升路径，提升公共决策的精细化水平。

**（5）促进农业研究的开放与协作。** 项目所形成的多模态数据集与系统 API 接口将开放给学术社区，促进跨学科、跨区域、跨年份的对比研究，推动农业风险研究的开放化与规范化。

## 9. 存在的问题和改进方向

### 9.1 模型层面

**问题。** 当前 Attention-LSTM 在原始单产上的 10 种子 R² 为 0.8160，仍有约 18% 的产量变异未被解释；模型对百年一遇极端事件的识别能力受限于样本规模（403 条）。

**改进方向。** （a）引入 Transformer 等更强的时序建模骨干；（b）通过 Bayesian Deep Learning 给出不确定性区间；（c）将 10 种子重复升级为 Nested CV，进一步收紧方差估计。

### 9.2 数据层面

**问题。** 研究目前仅覆盖省级面板；MODIS 遥感空间分辨率（250 m–1 km）偏粗放；西部气象站点密度不足；时间覆盖止于 2023 年。

**改进方向。** （a）下沉到地市级与县域级；（b）引入 Sentinel-2、高分系列高分辨率遥感；（c）加入土壤墒情、地下水位等新维度；（d）拓展时间跨度至 2024—2025 年。

### 9.3 系统层面

**问题。** 可视化系统目前以桌面端 Web 为主，移动端响应式适配仍有提升空间；情景模拟基于规则与模型重新推理，尚未引入因果推断框架。

**改进方向。** （a）开发响应式移动端与微信小程序；（b）引入 do-calculus、合成控制法等因果推断方法增强反事实情景的科学性；（c）增加多语种支持。

### 9.4 应用层面

**问题。** 韧性路径推荐目前基于 11 条规则引擎，缺乏与具体政策工具、财政预算的精细对接。

**改进方向。** （a）与地方农业部门、保险机构开展合作试点；（b）开发面向农户的简化版客户端；（c）探索系统在"互联网 + 农业保险""乡村振兴"等场景下的落地应用。
