01

整体流程总览

在动手之前，先看清楚这套工作流里的几个角色，知道每一步在干嘛，后面就不会迷路。

我们的协作大致由这几块拼起来，你会在不同环节用到它们：

Claude Project

团队在 claude.ai 上的项目空间，放着课题资料、报告、查新材料。和 Claude 讨论、写文档、查资料都在这里。

服务器

我们真正跑代码的地方。你用自己电脑上的 VS Code 远程连上去，在上面的「工作区」干活。

GitHub 仓库

xkwd-DC/DC-AIino 是代码的「云端总账本」。你已经加入了 xkwd-DC 组织，所以有权限。

Claude Code

装在服务器终端里的 AI 编程助手。它能读懂整个项目代码，帮你写代码、改 bug、跑命令。

把它串起来就是一条主线：

◷ 一句话流程

用 VS Code 远程连上服务器 → 在自己的工作区里 clone 仓库 → 在里面写代码并用 Claude Code 协助 → 用 Git 把成果 commit & push 回 GitHub。需要查资料、写报告时，就回到 Claude Project。

下面每一节都对应其中一步，照着做就行。完全没接触过命令行也别担心，需要敲的命令我都写好了，点「复制」即可。

02

在 Claude Project 里协作

Project 是我们这个课题的「共享大脑」——资料、上下文和对话都集中在一处，Claude 会基于这些内容回答你。

进入项目

  1. 打开 claude.ai 并用团队账号登录。
  2. 在左侧边栏点 Projects（项目），找到我们这个粮食生产风险可视化的课题项目并点进去。
  3. 项目里有两块：上方/右侧的 项目知识（Project knowledge） 放的是共享资料（论文、研究报告、查新报告、申报材料等）；中间是 对话区，每次新开一个聊天都会自动带上这些资料作为背景。

怎么用才顺手

  • 每个小任务开一个新对话。比如「帮我梳理核心因子识别的方法」和「帮我改研究报告第三章」分开聊，上下文更干净。
  • 需要 Claude 参考的资料就传进项目知识里，而不是每次粘贴一大段。传一次，之后所有对话都能用到。
  • 提问尽量说清背景和目标：你想要什么、给谁看、要多长、什么格式。要求越具体，结果越好。
  • 需要可下载的文件（Word 报告、PPT、Excel）时直接说，Claude 可以生成给你。
✦ 小提示

Project 里的对话只对项目成员可见，是放课题专属上下文的好地方。和别的不相干的事情（比如随便问个常识）就不必占用项目对话，单独开个普通聊天即可。

03

VS Code 连接服务器

用本地的 VS Code 远程连上服务器，就像在自己电脑上写代码一样，但文件和运行环境都在服务器上。你的 SSH 配置文件已经准备好了，所以这一步会很轻松。

第一次：装好插件

  1. 打开 VS Code，点左侧最下方的 扩展（Extensions）图标（四个小方块）。
  2. 搜索 Remote - SSH，安装由 Microsoft 出品的那个（或者直接装 Remote Development 扩展包，里面都包含了）。
  3. 装好后，VS Code 左下角会出现一个蓝绿色的 >< 远程按钮。

连接服务器

  1. 点左下角那个 >< 按钮，在弹出的菜单里选 Connect to Host…（连接到主机）。
  2. 因为 SSH 配置文件（~/.ssh/config）已经配好了，列表里会直接出现我们服务器的名字，点它就行，不用手动敲地址。
  3. 第一次连接可能会让你确认指纹（选 Continue）。如果之前配的是密钥认证，就不用输密码了；如果让你输密码，按提示输入即可。
  4. 连接成功后，左下角会显示 SSH: 服务器名，说明你现在已经「人在服务器上」了。

打开你的工作区

  1. 顶部菜单 File → Open Folder（打开文件夹）。
  2. 这时浏览的是服务器上的目录。进入分配给你的个人工作区文件夹（如果还没有，可以在终端里建一个，见下一节）。
  3. 顶部菜单 Terminal → New Terminal 打开终端，之后所有命令都在这个终端里敲——它直接跑在服务器上。
◷ 两种「SSH」别搞混

① 你的电脑 → 服务器：这个已经配好了，就是上面用的。
② 服务器 → GitHub：这是另一回事，是从服务器拉取/推送代码用的认证，下一节会单独说。

04

在工作区 clone 仓库

「clone（克隆）」就是把 GitHub 上的仓库完整下载一份到服务器上，变成你能编辑的本地文件夹。你已经在 xkwd-DC 组织里，所以有读写权限。

① 先告诉 Git 你是谁（只需一次）

这一步是给你之后的每一次提交署名，全局设置一次即可。在终端里运行：

bash · 在服务器终端
$ git config --global user.name "你的名字"
$ git config --global user.email "你的GitHub邮箱"

② 让服务器能访问 GitHub（认证）

从服务器拉代码需要让 GitHub 认识这台机器。推荐用 SSH 密钥方式，配一次以后都不用输密码：

bash · 生成并查看密钥
# 1) 生成一对密钥（一路回车即可，邮箱换成你的）
$ ssh-keygen -t ed25519 -C "你的GitHub邮箱"

# 2) 打印出公钥，整段复制下来
$ cat ~/.ssh/id_ed25519.pub
  1. 复制上面打印出来的公钥（以 ssh-ed25519 开头的一整行）。
  2. 打开 GitHub → 右上角头像 → Settings → SSH and GPG keys → New SSH key，把公钥粘进去保存。
  3. 验证一下是否连通：
bash · 测试连接
$ ssh -T git@github.com
# 看到 "Hi 你的用户名! You've successfully authenticated" 就成功了
◷ 不想配 SSH？

也可以用 HTTPS + 访问令牌（PAT）：clone 时地址换成 https://github.com/...，遇到要密码时，用 GitHub 个人访问令牌代替密码（在 Settings → Developer settings → Personal access tokens 里生成）。SSH 配一次更省心，按需选择。

③ Clone 仓库到你的工作区

bash · 克隆代码
# 进入（或新建）你的个人工作区目录
$ mkdir -p ~/workspace # 如果还没有就建一个
$ cd ~/workspace

# 用 SSH 方式克隆（推荐）
$ git clone git@github.com:xkwd-DC/DC-AIino.git

# 或者用 HTTPS 方式克隆
$ git clone https://github.com/xkwd-DC/DC-AIino.git

# 进入项目文件夹
$ cd DC-AIino

完成后，在 VS Code 里 File → Open Folder 选中这个 DC-AIino 文件夹，就能开始编辑了。

⚠ 开干之前先拉一下

每次开始工作前，先 git pull 把别人最新的改动同步下来，能减少很多冲突。这个习惯很值钱。

05

Git / GitHub 常用操作

Git 是给代码做「存档」的工具。先理解几个核心概念，命令就很好记了。

先搞懂三个空间

  • 工作区：你正在编辑的文件本身。
  • 暂存区（staged）：你挑出来「准备这次要存档」的改动，用 git add 放进去。
  • 本地仓库：用 git commit 把暂存区的改动正式存成一个「快照」，留在你这台机器上。
  • 远程仓库（GitHub）：用 git push 把本地的 commit 上传到 GitHub，别人才看得到。
✦ 关键词翻译

commit（提交）＝ 在本地给当前改动拍一张存档照片，附一句说明。
push（推送）＝ 把这些存档照片上传到 GitHub。
pull（拉取）＝ 把 GitHub 上别人的最新改动下载到本地。
简单记：commit 是存档，push 是上传，pull 是同步。

日常工作流（最常用的一套）

bash · 一天的典型流程
# 1) 开工前，先同步最新代码
$ git pull

# 2) 改完文件后，看看动了哪些
$ git status

# 3) 把要提交的改动放进暂存区（. 表示全部）
$ git add .

# 4) 提交一个存档，写清楚这次干了啥
$ git commit -m "完成核心因子识别模块的数据预处理"

# 5) 上传到 GitHub，让队友看到
$ git push

用分支（branch）来不互相干扰

多人协作时，最好各自开一条分支干活，做完再合并回主分支，避免直接改乱主线。

bash · 分支操作
# 新建并切换到一条自己的分支
$ git checkout -b feature/data-cleaning

# 在分支上正常 add / commit / push
$ git push -u origin feature/data-cleaning

# 切回主分支
$ git checkout main

推上去后，可以在 GitHub 网页上发起 Pull Request（PR），让队友 review 后再合并进 main。这是最稳妥的协作方式。

常用命令一览

命令作用
git status查看当前有哪些改动、在哪条分支
git add .把所有改动放进暂存区（也可指定单个文件）
git commit -m "说明"把暂存区的改动存为一个本地快照
git push把本地 commit 上传到 GitHub
git pull把 GitHub 上的最新改动同步到本地
git log --oneline简洁地查看提交历史
git diff查看具体改了哪几行
git branch列出所有分支（带 * 的是当前分支）
git checkout -b 名字新建并切换到一条新分支
git clone 地址把远程仓库下载到本地
⚠ 别慌的几个原则

commit 信息写清楚「做了什么」，方便日后回溯；不确定操作前先 git status 看清状态；遇到 push 被拒（别人先推了），先 git pull 再 push。真遇到冲突或搞不定，先停手别乱试，截图发群里问。

06

启动并使用 Claude Code

Claude Code 是跑在终端里的 AI 编程助手。它能读懂整个项目、帮你写和改代码、还能直接运行命令。下面在服务器终端里完成安装和启动。

① 安装

最通用的方式是用 npm 安装（需要服务器上已装好 Node.js 18 及以上）：

bash · 安装 Claude Code
# 先确认 Node 版本（应 ≥ 18）
$ node --version

# 全局安装 Claude Code（不要加 sudo）
$ npm install -g @anthropic-ai/claude-code

# 验证安装成功
$ claude --version
◷ 装不上 / 权限报错？

如果遇到 EACCES 权限错误，不要用 sudo。把 npm 全局目录改到自己有权限的位置即可：
mkdir ~/.npm-global → npm config set prefix '~/.npm-global' → 把 export PATH="$HOME/.npm-global/bin:$PATH" 加到 ~/.bashrc 或 ~/.zshrc，再 source 一下重新安装。Anthropic 也提供「原生安装器」（无需 Node.js），详见 官方文档。

② 启动并登录

  1. 进入项目目录（很重要——它会以当前文件夹为工作上下文）：cd ~/workspace/DC-AIino
  2. 输入 claude 启动。首次运行会引导你登录授权。
  3. 按提示用团队的 Claude 账号完成登录（会给一个链接/验证码的授权流程），授权后凭证保存在本地，之后就不用再登了。
bash · 启动
$ cd ~/workspace/DC-AIino
$ claude

③ 怎么用

启动后进入一个对话式界面，你直接用大白话描述需求就行，比如：

  • 「读一下这个项目，告诉我整体结构和各模块是干嘛的。」
  • 「帮我在 data/ 目录写一个脚本，把气候数据按月份聚合。」
  • 「这段代码报了这个错，帮我定位并修好，然后跑测试验证。」

Claude Code 会自己读相关文件、提出修改、（在你允许后）执行命令。第一次进新项目，建议先让它生成一份项目说明文件：

claude · 斜杠命令
/init      # 自动生成 CLAUDE.md（记录项目命令、结构、约定）
/help      # 查看所有可用命令
/clear     # 清空当前上下文，开始新任务
/compact   # 压缩对话历史，省 token
✦ CLAUDE.md 是个好东西

/init 生成的 CLAUDE.md 会被 Claude 每次开场自动读取，相当于项目的「使用说明」。把它一起 commit 到仓库，全组都能共享同一套上下文，Claude 对项目的理解会更准。

07

安装 Claude Code 技能（Skills）

技能（Skill）是一包包「专项本领」——比如处理 Word/PDF/PPT、做前端设计等。装上之后，Claude Code 在相关任务里会自动调用，特别适合我们这种又要写报告、又要做可视化系统的课题。

从官方仓库安装（推荐）

在 Claude Code 会话里（也就是启动 claude 之后），输入下面的斜杠命令：

claude · 安装官方技能
# 1) 把 Anthropic 官方技能仓库注册为「插件市场」
/plugin marketplace add anthropics/skills

# 2) 安装文档处理技能（Word / PDF / PPT / Excel）
/plugin install document-skills@anthropic-agent-skills

# 3) 安装示例/常用技能合集
/plugin install example-skills@anthropic-agent-skills

# 4) 查看已装好的技能
/skills

装好后，不用做额外配置，在提问里提到相关任务就会自动触发，比如：「用 PDF 技能把这份查新报告里的表格提取出来」「用 docx 技能帮我把研究报告排成正式的 Word 文档」。

从任意 GitHub 仓库手动安装

如果想装某个 GitHub 上的开源技能，可以直接把它放进技能目录：

bash · 手动安装
# 个人级（对所有项目生效）：克隆进 ~/.claude/skills/
$ git clone https://github.com/作者/技能名.git ~/.claude/skills/技能名

# 项目级（只对当前仓库生效）：放进项目里的 .claude/skills/
$ git clone https://github.com/作者/技能名.git .claude/skills/技能名
⚠ 三个常见坑

① SKILL.md 必须在技能文件夹的第一层（路径形如 ~/.claude/skills/技能名/SKILL.md），不能再嵌一层。
② 安装非官方技能前，先大致看一眼它的内容，确认来源可信再用。
③ 装完用 /skills 确认是否加载成功；没出现就检查路径和 SKILL.md 的格式。

✦ 我们项目适合先装哪些

报告/查新材料多 → document-skills（Word/PDF/PPT/Excel）很实用；要做「可视化系统」的界面 → 可以找 frontend-design 相关技能。先把这两类装上，基本够用了。
