SECTION · 01

一、设计原则

三条贯穿全局的原则

★ 原则 1 · 数据先行
每一像素都为数据服务。装饰元素永远不能抢走数据图表的焦点;高亮、动效、色彩仅用于强调"读者需要看见的那个数字"。
★ 原则 2 · 学术克制
这是面向政府/农险/科研机构的工具,不是消费产品。所有视觉元素须传递"严谨、可信、专业"——避免渐变滥用、避免拟物、避免感性辞令。
★ 原则 3 · 信息层级清晰
每一屏都要有明确的"第一眼/第二眼/第三眼"。用字号、色彩亮度、留白三个维度建立层级,而不是用边框和分隔线堆砌。

七、提示与通知

四种 callout 共用相同结构,通过左边框颜色区分语义。标题使用 mono 字体小型大写,正文使用 13px 普通正文。

★ 默认 · 强调说明

✕ 危险 · DANGER
用于高风险结论或破坏性操作前的警告。例如:该路径推荐将覆盖现有方案,操作不可撤销。

十二、动效规范

动效服从"克制 + 有意义"两条准则。所有动效都为"传递信息"或"引导注意",从不为炫技。后续模块只能从本规范中调用动效,不要自行新增。

核心 Token

DURATION · 时长
Token时长用途
--dur-fast150ms点击反馈 / 按下态 / 即时响应
--dur-base250ms常规 hover / 颜色过渡 / 默认
--dur-slow450ms卡片展开 / 进度条填充
--dur-entrance600ms页面/章节入场
EASING · 缓动函数
Token曲线用途
--ease-outcubic-bezier(.22, 1, .36, 1)默认 · 元素进入
--ease-in-outcubic-bezier(.65, 0, .35, 1)呼吸 / 循环类
--ease-springcubic-bezier(.34, 1.56, .64, 1)弹性反馈 · 按钮 / 数字 / 徽章

动效分级

L1 · 入场动效(一次性)

  • Hero 内部元素 stagger 出现(eyebrow → h1 → sub → stats),delay 间隔 80-120ms
  • 滚动进入视口的 section 整体上移淡入(IntersectionObserver 触发)
  • 关键数字 count-up(数据先模糊后清晰)

L2 · 交互反馈(用户触发)

  • 按钮 hover:上移 1px + 阴影扩大;active:下沉 1px
  • 卡片 hover:上移 2-3px + 阴影 + 顶部进度线扫入
  • 表格行 hover:背景渐变 + 左侧绿色指示条
  • 色卡 hover:光泽扫过(700ms)
  • 滑块 thumb hover:放大至 18px

L3 · 环境动效(持续 · 低强度)

  • Hero 背景径向光斑缓慢漂移(22-28s 周期)
  • Header 在线状态点呼吸脉冲(2.4s)
  • 滑块填充条扫光(4s linear)
  • code block 顶部扫描线(5s linear)
  • code lang 标签的 ▸ 闪烁(模拟终端光标)
⚠ 环境动效红线
L3 类动效频率必须 ≥ 2 秒,亮度差 ≤ 30%,不能形成视觉脉冲干扰。任何一屏的环境动效不超过 3 个,数据图表周围禁用环境动效。

无障碍 · prefers-reduced-motion

本系统已实现媒体查询 @media (prefers-reduced-motion: reduce):用户在系统中开启"减少动态效果"时,所有动效自动降至 0.01ms。后续模块必须沿用此机制,不可硬编码强制动效。

CSS · 必备片段
@media (prefers-reduced-motion: reduce) {
  *, *::before, *::after {
    animation-duration: 0.01ms !important;
    animation-iteration-count: 1 !important;
    transition-duration: 0.01ms !important;
  }
}

/* 滚动入场:用 IntersectionObserver 给元素加 .in-view */
.reveal {
  opacity: 0;
  transform: translateY(24px);
  transition: opacity 600ms var(--ease-out),
              transform 600ms var(--ease-out);
}
.reveal.in-view {
  opacity: 1;
  transform: translateY(0);
}
✕ 动效禁区
  • 不用 bounce / shake / wiggle 这类幼稚动效
  • 数据图表本身不加入场动画(ECharts 默认动画足够,不要叠加)
  • 不在 hover 上做超过 300ms 的动画(用户会等不及)
  • 不用持续旋转的 loading,改用 shimmer 占位
  • 不用视差滚动(parallax),严肃报告类应用不需要
SECTION · 13

十三、使用规范

给后续 4 个模块的硬性约定

✓ 必做
  • 所有颜色通过 CSS 变量调用,不硬编码 hex
  • 所有数字、英文、时间戳使用 JetBrains Mono
  • 中文一级/二级标题使用 Noto Serif SC
  • 每个模块顶部保留 sticky header + 品牌区
  • 每张数据图表必须有标题 + 副标题 + 数据来源
✕ 禁用
  • 不用纯黑 (#000) 作背景,使用 --bg
  • 不用紫色渐变 / 霓虹色 / 拟态阴影
  • 不用 Inter / Roboto / 系统默认字体
  • 不用圆角超过 8px(只有 hero 容器允许 8px)
  • 不用 emoji 替代图标

跨模块视觉一致性检查清单

  1. 4 个模块的 header 高度、品牌区位置是否一致?
  2. 卡片圆角是否统一为 6px?
  3. 所有数字字号、字重、颜色是否一致?
  4. 图表配色是否全部走 CSS 变量?
  5. callout / badge / button 是否完全复用本系统?
  6. 1920×1080 视口下没有水平滚动?
